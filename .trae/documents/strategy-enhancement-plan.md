# 游戏策略性增强方案

## 一、当前问题分析

### 1.1 策略性不足的核心问题

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 第一关目标太少（仅钥匙） | 记忆槽取舍感不强 | 🔴 高 |
| 混乱值增长太慢 | 没有时间压力和紧迫感 | 🔴 高 |
| 记忆过期惩罚轻 | 不更新记忆代价低 | 🟡 中 |
| 事件触发太机械 | 玩家行为不影响事件 | 🟡 中 |
| 记忆槽缺乏差异化策略 | 所有记忆优先级相同 | 🟡 中 |

### 1.2 当前数值配置（levelBalance.ts）

```typescript
chaosGrowthPerSecond: 0.15,  // 180秒自然增长仅27，压力感不足
wrongPlacementChaos: 8,       // 惩罚太轻
outdatedMemoryChaos: 4,       // 过期记忆惩罚几乎没有
memoryUpdateScore: 30,        // 更新记忆奖励太少
```

## 二、策略性增强方案

### 2.1 方案概览

| 模块 | 增强方向 | 核心改动 |
|------|----------|----------|
| **记忆系统** | 增加策略取舍 | 记忆槽优先级差异化、过期惩罚增强、覆盖选择机制 |
| **混乱值系统** | 增加压力感 | 提高增长率、增加机制性影响（不只是视觉） |
| **任务设计** | 增加目标数量 | 第一关恢复3个目标、增加任务复杂度 |
| **事件系统** | 增加动态性 | 基于混乱值的随机事件、玩家行为反馈 |
| **评分系统** | 奖励策略性操作 | 记忆优化奖励、时间管理奖励 |

### 2.2 具体改动设计

#### 2.2.1 记忆系统策略增强

**目标**：让玩家在记忆管理上做出有意义的选择

**改动1：记忆优先级差异化**

```typescript
// 新增记忆类型权重
const MEMORY_TYPE_WEIGHTS = {
  spatial: { scoreBonus: 50, chaosReduction: 5 },     // 空间记忆价值最高
  object: { scoreBonus: 30, chaosReduction: 3 },     // 物体记忆中等
  temporal: { scoreBonus: 40, chaosReduction: 4 },   // 时序记忆较高
  procedural: { scoreBonus: 60, chaosReduction: 6 }, // 程序记忆价值最高
}

// 记忆过期惩罚增强
outdatedMemoryChaos: 15,  // 从4提升到15
outdatedMemoryPenalty: 30, // 新增：过期记忆被使用时扣分
```

**改动2：记忆覆盖选择机制**

- 当记忆槽满时，弹出覆盖选择界面
- 显示每个记忆的优先级、置信度、是否过期
- 玩家必须主动选择覆盖哪个记忆
- 覆盖高优先级记忆会有惩罚提示

**改动3：记忆连锁反应**

- 关键物品（钥匙、手机）的记忆过期时，连锁标记相关记忆过期
- 例如：钥匙被移动，同时标记"玄关托盘"的记忆为半过期

#### 2.2.2 混乱值系统压力增强

**目标**：让混乱值真正影响游戏机制，增加紧迫感

**改动1：提高混乱值增长率**

```typescript
chaosGrowthPerSecond: 0.3,  // 从0.15提升到0.3
```

**改动2：混乱值机制性影响**

| 混乱值范围 | 视觉效果 | 机制效果 |
|-----------|----------|----------|
| 0-30% | 正常 | 无额外影响 |
| 30-60% | 轻微暗角 | 记忆过期速度加快1.5倍 |
| 60-80% | 明显暗角+噪点 | 记忆过期速度加快2倍，随机事件概率提升 |
| 80-100% | 严重视觉干扰 | 记忆过期速度加快3倍，物体随机移动 |

**改动3：混乱值惩罚机制**

- 混乱值 > 60%：每次操作前有概率触发"记忆模糊"效果
- 混乱值 > 80%：每10秒随机移动一个自由物品
- 混乱值 = 100%：游戏失败（已实现）

#### 2.2.3 任务设计增强

**目标**：增加目标数量和任务复杂度，体现策略取舍

**改动1：第一关恢复3个目标**

恢复原设计：钥匙、手机、雨伞三个目标

**改动2：目标优先级差异化**

```typescript
goals: [
  {
    id: 'g-key-on-tray',
    description: '钥匙放到玄关托盘',
    priority: 'high',      // 高优先级：必须完成
    memoryType: 'spatial',
    ...
  },
  {
    id: 'g-phone-on-tray',
    description: '手机放到玄关托盘',
    priority: 'medium',    // 中优先级：重要但可稍后
    memoryType: 'object',
    ...
  },
  {
    id: 'g-umbrella-on-tray',
    description: '雨伞放到玄关托盘',
    priority: 'low',       // 低优先级：可选目标，加分项
    memoryType: 'spatial',
    ...
  },
]
```

**改动3：目标依赖关系**

- 某些目标需要先完成前置目标
- 例如：必须先拿到钥匙才能打开某个抽屉拿到手机

#### 2.2.4 事件系统动态化

**目标**：让事件基于玩家行为和混乱值动态触发

**改动1：基于混乱值的随机事件**

```typescript
// 新增：混乱值驱动的随机事件
const CHAOS_EVENTS = [
  {
    id: 'random-hide',
    description: '某个物品被藏起来了',
    minChaos: 50,
    probability: 0.02, // 每2秒2%概率
    effect: 'hide-random-entity',
  },
  {
    id: 'random-move',
    description: '某个物品被移动了',
    minChaos: 70,
    probability: 0.03,
    effect: 'move-random-entity',
  },
]
```

**改动2：玩家行为反馈事件**

- 连续3次错误操作 → 触发"烦躁"事件，混乱值大幅增加
- 连续5次正确操作 → 触发"专注"事件，混乱值下降
- 长时间不保存记忆 → 触发"记忆警告"事件

#### 2.2.5 评分系统优化

**目标**：奖励策略性操作，惩罚盲目操作

**改动1：记忆策略评分**

```typescript
// 新增评分项
memoryEfficiencyBonus: 100,       // 记忆有效率达到80%以上奖励
memoryOptimizationBonus: 50,      // 更新过期记忆奖励
memoryLockBonus: 30,              // 锁定关键物品记忆奖励
```

**改动2：时间管理评分**

```typescript
timeBonusRate: 3,                 // 从2提升到3
earlyCompletionBonus: 200,        // 提前完成目标奖励
```

**改动3：Combo系统增强**

```typescript
maxComboMultiplier: 3.0,          // 从2提升到3
comboBreakPenalty: 30,            // 中断Combo惩罚
```

## 五、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 数值调整导致关卡难度过高 | 玩家无法通关 | 保留原配置，提供难度选择 |
| 记忆系统增强导致复杂度增加 | 新玩家难以理解 | 增加详细教程和提示 |
| 随机事件导致游戏不可预测 | 玩家体验不稳定 | 事件概率可控，保留可复现性 |
| 连锁过期导致挫败感 | 玩家频繁丢失记忆 | 关键记忆有保护机制 |
