# 三大提升点实施计划

## 一、移动端适配

### 1.1 现状分析

当前控制完全依赖键盘（WASD移动、鼠标旋转视角），无任何触摸支持。HUD 布局固定尺寸，在小屏幕上会重叠或溢出。

### 1.2 技术方案

**A. 虚拟摇杆控制**
- 创建 `VirtualJoystick.tsx` 组件，支持触摸拖拽
- 摇杆区域固定在屏幕左下角
- 摇杆输出归一化向量（x, y），映射到移动方向

**B. 视角旋转控制**
- 屏幕右侧区域作为视角控制区
- 单指拖拽：旋转视角（yaw/pitch）
- 双指捏合：缩放（可选，用于顶视图）

**C. 交互按钮**
- 屏幕右下角放置交互按钮（拾取/放置/打开）
- 记忆保存按钮（原 E 键功能）
- 视图切换按钮（原 V 键功能）

**D. 响应式 HUD**
- 使用 Tailwind CSS 断点（sm, md）调整布局
- 小屏幕下记忆槽改为横向排列底部
- 任务面板改为可折叠抽屉式

### 1.3 修改文件

| 文件 | 修改内容 |
|------|----------|
| `src/components/arena3d/FirstPersonControls.tsx` | 添加触摸事件处理，支持虚拟摇杆输入 |
| `src/components/arena3d/HUD.tsx` | 添加响应式布局，隐藏桌面端按钮 |
| `src/components/arena3d/VirtualJoystick.tsx` | **新建** 虚拟摇杆组件 |
| `src/game/playerControls.ts` | 添加触摸输入类型支持 |
| `src/store/slices/playerSlice.ts` | 添加触摸状态管理 |

### 1.4 风险与注意事项

- 触摸事件与鼠标事件冲突：使用 `PointerEvent` 统一处理
- 移动端性能：降低渲染分辨率、禁用阴影
- 屏幕方向：适配横屏和竖屏

## 二、音效系统完善

### 2.1 现状分析

当前音效系统基于 Web Audio API 生成合成音效，缺少：
- 背景音乐（BGM）
- 环境音效（脚步声、开门声、水流声等）
- 3D 音效定位

### 2.2 技术方案

**A. 背景音乐系统**
- 使用 AudioBuffer 加载预生成的 BGM（每个关卡不同）
- 添加 BGM 音量控制（独立于 SFX）
- 支持 BGM 淡入淡出切换

**B. 环境音效**
- 添加环境音效循环（房间特有：客厅、卧室、厨房）
- 添加动态音效：脚步声（根据移动速度）、开门声

**C. 3D 音效**
- 使用 Web Audio API 的 PannerNode 实现空间音效
- 根据物体位置调整音效方向和距离衰减

### 2.3 修改文件

| 文件 | 修改内容 |
|------|----------|
| `src/audio/sfx.ts` | 添加 BGM 系统、环境音效、3D 音效支持 |
| `src/audio/bgm.ts` | **新建** BGM 配置和管理 |
| `src/components/arena3d/Scene3D.tsx` | 根据房间切换环境音效 |
| `src/game/commands.ts` | 添加开门/脚步声触发 |
| `src/components/arena3d/HUD.tsx` | 添加 BGM 音量控制按钮 |

### 2.4 风险与注意事项

- 文件大小：BGM 使用短循环（15-30秒），压缩为 OGG 格式
- 浏览器兼容性：部分移动端浏览器不支持某些音频格式
- 性能：使用 AudioWorklet 处理复杂音效

## 三、对话系统开发

### 3.1 现状分析

当前剧情仅通过 toast 提示展示，无对话框、无角色头像、无打字机效果，沉浸感不足。

### 3.2 技术方案

**A. 对话框组件**
- 创建 `DialogBox.tsx`，包含角色头像、名字、对话内容
- 支持打字机效果（逐字显示）
- 支持快速跳过（点击屏幕）

**B. 对话数据结构**
- 定义 `DialogLine` 接口：角色ID、头像、文本、情绪
- 定义 `DialogSequence`：多条对话组成的序列
- 支持分支对话（简单选项）

**C. 与现有系统集成**
- 脚本事件触发对话序列
- 关卡开始/结束时显示剧情对话
- 记忆小妖事件触发对话

### 3.4 风险与注意事项

- 性能：打字机效果使用 requestAnimationFrame，避免阻塞
- 国际化：预留多语言支持接口
- 存档：对话状态需要保存到 session

## 四、实施顺序与优先级

### Phase 1（基础）：音效系统完善
- 预计工时：中等
- 风险：低
- 收益：高（提升沉浸感）

### Phase 2（核心）：移动端适配
- 预计工时：较高
- 风险：中等
- 收益：极高（扩大用户群）

### Phase 3（体验）：对话系统开发
- 预计工时：中等
- 风险：低
- 收益：高（提升剧情沉浸感）

### 推荐实施顺序：Phase 1 → Phase 2 → Phase 3

## 七、潜在问题与应对

1. **移动端性能问题**
   - 应对：降低渲染质量、禁用阴影、使用 LOD

2. **音频文件体积过大**
   - 应对：使用 OGG 压缩格式、循环播放

3. **对话文本过长**
   - 应对：分段显示、添加滚动支持

4. **触摸控制误操作**
   - 应对：添加触摸区域边界检测、长按确认
