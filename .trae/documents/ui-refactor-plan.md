# UI 界面优化计划

## 问题分析

根据用户反馈，当前游戏界面存在以下问题：

1. **底部蓝色条**：Footer 显示"HomeMem Arena — AI 创造力比赛 Demo · 面向家务机器人记忆能力研究"，破坏游戏沉浸感
2. **任务简报排版**："早餐时间循环"的简报内容排版混乱，需要重新设计
3. **文字颜色看不清**：简报中"任务进行中会有脚本事件触发..."这行文字颜色太暗
4. **HUD 界面乱糟糟**：左侧控制面板（检查完成度、前往记忆测试、重新开始）遮挡任务列表，整体布局不合理

## 修复方案

### 1. 移除底部 Footer
- 修改 Footer.tsx，移除或简化版权信息
- 在 ArenaPage 中隐藏 Footer（游戏页面不需要底部栏）

### 2. 优化任务简报样式
- 重新设计任务简报卡片，使用游戏风格的视觉设计
- 将简报内容解析为结构化展示（标题、规则列表、陷阱列表）
- 使用更清晰的颜色方案，提高文字可读性
- 添加视觉分隔符和图标增强层次

### 3. 优化 HUD 布局
- 将左侧控制面板移到更合理的位置（如右下角或整合到任务面板中）
- 移除冗余或研究平台风格的按钮
- 优化任务列表的显示方式

### 4. 更新关卡简报内容排版
- 更新 breakfast.ts 中的 briefing 字段，使用更清晰的格式

## 修改文件清单

| 文件 | 修改内容 |
|------|---------|
| `src/components/layout/Footer.tsx` | 移除版权信息或改为游戏风格 |
| `src/pages/ArenaPage.tsx` | 优化任务简报样式，调整控制面板位置 |
| `src/data/tasks/breakfast.ts` | 优化 briefing 内容排版 |
| `src/components/arena3d/HUD.tsx` | 优化任务列表显示，整合控制面板 |

## 实施步骤

1. 首先修改 Footer.tsx，移除底部蓝色条
2. 然后优化 ArenaPage.tsx 中的任务简报样式
3. 更新 breakfast.ts 的 briefing 内容
4. 最后优化 HUD 的布局和控制面板

## 风险评估

- 低风险：所有修改都是 UI 层面的，不影响核心游戏逻辑
- 需要注意：确保修改后游戏功能仍然正常，特别是任务完成检测和记忆测试跳转
