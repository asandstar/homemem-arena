# 第一关修复计划

## 问题分析

从截图和代码分析，第一关存在以下核心问题：

### 1. 位置计算错误（严重）

在 `placement.ts` 的 `getFreeObjectInitialPosition` 函数中：
```typescript
const baseX = room.center.x + objSpec.initialPosition.x
const baseZ = room.center.z + objSpec.initialPosition.z
```

但任务配置中的坐标被错误地当作世界坐标而非相对偏移量。

**餐厅房间信息**：
- 中心：{ x: 16, y: 0, z: 0 }
- 大小：{ x: 8, y: 3, z: 8 } → x 范围：12~20，z 范围：-4~4

**当前错误计算**：
- 脏杯子：x = 16 + 15.4 = 31.4（远超出餐厅）
- 洗碗机：x = 16 + 18.5 = 34.5（远超出餐厅）
- 垃圾桶：x = 16 + 13.5 = 29.5（远超出餐厅）

### 2. 光照太暗（严重）

从截图看场景几乎全黑，物品不可见：
- 环境光强度只有 0.15
- 餐厅点光强度只有 0.55
- 非当前房间光照被降低到 15%

### 3. 模型显示异常（中等）

截图中显示多个杯子重复出现，可能是模型生成或渲染问题。

## 修复方案

### 步骤 1：修复第一关坐标

重新计算相对于餐厅中心的偏移量：

**容器位置调整**（餐厅中心 x=16, z=0）：
- 餐桌：{ x: 0, y: 0.45, z: 0 }（中心）
- 洗碗机：{ x: 2.5, y: 0.4, z: 0 }（右侧）
- 垃圾桶：{ x: -2.5, y: 0.2, z: 0 }（左侧）
- 餐具架：{ x: -1.5, y: 0.4, z: 0 }（左侧偏中）

**物品位置调整**（在餐桌上）：
- 脏杯子：{ x: -0.6, y: 0, z: 0 }（餐桌左侧）
- 餐巾纸：{ x: 0.6, y: 0, z: 0 }（餐桌右侧）
- 叉子：{ x: 0, y: 0, z: -0.3 }（餐桌前方）

### 步骤 2：调整光照系统

增加全局环境光和餐厅光照强度：
- 全局环境光：0.15 → 0.35
- 餐厅点光强度：0.55 → 0.8
- 非当前房间光照保留比例：0.15 → 0.3

### 步骤 3：验证修复

1. 运行测试确保位置计算正确
2. 启动开发服务器验证视觉效果
3. 测试物品拾取和放置功能

## 风险评估

- **低风险**：坐标调整是数值修正，不影响逻辑
- **中等风险**：光照调整可能影响其他关卡的视觉效果，需要验证

## 验证标准

1. 物品正确显示在餐厅内的餐桌上
2. 容器（洗碗机、垃圾桶、餐具架）正确显示在餐厅内
3. 场景亮度足够，物品清晰可见
4. 物品可以正常拾取和放置
