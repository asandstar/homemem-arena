# 新手引导轻量级改进计划

## 一、调研结论

### 当前现状
游戏已有基础的提示机制：任务简报、操作提示面板、任务目标面板、近距离高亮、悬停提示、Toast 反馈、记忆槽界面、小地图。

### 新手主要困惑点（按优先级）
1. **记忆系统不清晰** - 为什么保存？过期/置信度/锁定是什么意思？和任务的关系？
2. **混乱值系统说明不足** - 怎么产生的？高了会怎样？怎么降低？
3. **交互方式容易混淆** - 点击 vs F 键？手持物品怎么放？怎么从容器拿东西？
4. **缺少游戏内帮助** - 玩到一半想查操作说明或游戏规则没地方看
5. **首次进入无引导** - 新玩家一上来就面对复杂界面，学习曲线陡

### 技术栈
- React + TypeScript
- Zustand 状态管理
- Tailwind CSS 样式
- React Three Fiber 3D 渲染

---

## 二、改进目标

通过轻量级改动，让新手玩家能：
1. 随时查阅游戏系统说明（帮助面板）
2. 理解记忆系统的作用和机制
3. 理解混乱值的影响和应对
4. 第一次进入时有基本的方向感

---

## 三、具体功能设计

### 功能 1：游戏内帮助面板（按 H 键）

**功能描述**：
- 游戏中按 H 键打开/关闭帮助面板
- 用标签页分类展示：操作说明、记忆系统、混乱值、任务目标
- ESC 键也可以关闭

**内容规划**：

| 标签页 | 内容 |
|--------|------|
| 🎮 操作说明 | 所有快捷键 + 交互方式说明（点击/F键、拾取放置逻辑） |
| 🧠 记忆系统 | 为什么保存记忆、置信度/过期/锁定的含义、记忆怎么帮你完成任务 |
| ⚠️ 混乱值 | 混乱值如何产生、高混乱值的后果、如何降低混乱值 |
| 🎯 游戏目标 | 怎么算完成任务、评分机制、Combo 系统 |

**实现方式**：
- 新建 `HelpPanel.tsx` 组件
- 在 HUD 中渲染（pointer-events-auto）
- 用 `useState` 控制开关
- 监听键盘 H 键和 ESC 键

---

### 功能 2：记忆系统说明 tooltip

**功能描述**：
- 记忆槽标题旁边加一个问号按钮
- 鼠标悬停或点击时显示记忆系统简要说明
- 点击"了解更多"跳转到帮助面板的记忆系统标签页

**实现方式**：
- 在 HUD 的记忆槽区域添加问号图标按钮
- 用简单的悬浮卡片展示说明
- 点击问号时打开帮助面板并切到记忆系统标签

---

### 功能 3：混乱值说明 tooltip

**功能描述**：
- 鼠标悬停混乱值进度条时显示说明
- 说明混乱值的含义、增长原因、降低方法

**实现方式**：
- 在混乱值条外层加一个 hover 触发的 tooltip
- 用 CSS 或 state 控制显示

---

### 功能 4：首次进入欢迎引导

**功能描述**：
- 检测玩家是否第一次进入游戏（localStorage）
- 第一次进入时显示一个简洁的欢迎弹窗
- 告诉玩家：这是什么游戏、基本目标、按 H 可以随时看帮助
- 有"开始游戏"和"先看教程"两个按钮

**实现方式**：
- 在 `ArenaPage.tsx` 或首页添加首次访问检测
- 用 `localStorage` 存储 `hasSeenWelcome` 标记
- 欢迎弹窗风格与任务简报一致

## 六、UI 设计要点

- **风格统一**：与现有 HUD 风格一致（深色半透明背景、毛玻璃效果、紫色强调色）
- **位置合理**：帮助面板居中显示，不阻挡重要游戏元素
- **操作便捷**：H 键开关 + ESC 关闭 + 点击遮罩关闭
- **内容清晰**：用图标 + 小标题 + 简短说明，避免大段文字
- **层级正确**：帮助面板 z-index 高于游戏场景，低于异常事件弹窗

---

## 七、风险与注意事项

### 潜在风险
1. **内容过多** - 帮助面板内容太复杂，新手更困惑
   - 应对：用简洁的要点式说明，避免大段文字
   
2. **影响沉浸感** - 帮助面板打断游戏节奏
   - 应对：面板半透明 + 动画过渡流畅 + 随时可关闭

3. **状态管理复杂** - 多个地方控制帮助面板开关
   - 应对：用本地 useState 管理即可，不需要进全局 store

### 兼容性
- 不影响现有游戏逻辑
- 不修改现有状态结构
- 纯新增 UI 组件
