# Core Interaction Stability Pass 修复计划

## 调研结论

通过代码审查，发现以下 4 大类问题需要修复：

### 问题 1：物体/家具摆放悬空
**现状分析：**
- `Room3D.tsx` 中家具装饰的 Y 轴位置是硬编码的，很多家具 group 的 y 坐标设置为 `size.y / 2`，但实际几何模型的原点可能不在底部
- `FurnitureModel.tsx` 中有 `position={[0, -visualHeight / 2, 0]}` 的偏移，意味着模型原点在中心，需要向下偏移一半才能落地
- `RoomDecorations` 中很多家具的 Y 位置可能没有正确对齐地面
- `placement.ts` 中 `snapEntityToWorld` 对普通物体有贴地计算，但家具是硬编码位置

**具体问题点：**
- 客厅地毯 y=0.02 ✅ 正确
- 沙发 y=0.45，size.y=0.9 → 中心在 0.45，底部在 0 → ✅ 正确
- 鞋柜 y=0.6，size.y=1.0 → 底部在 0.1，悬浮 0.1m ❌
- 植物 y=0.7，size.y=0.6 → 底部在 0.4，悬浮严重 ❌
- 台灯 y=0.9，size.y=1.8 → 底部在 0 → ✅ 正确
- 等等...需要逐一审视

### 问题 2：玩家移动不流畅
**现状分析：**
- `FirstPersonControls.tsx` 中使用 `useFrame` 每帧直接 setState 更新位置
- 位置更新有阈值判断（`dx * dx + dz * dz > 0.000001`），但相机是每帧直接跟随
- 缺少移动插值/平滑处理
- 鼠标视角控制是直接设置 `robotRotation`，没有平滑过渡

**具体问题：**
- 键盘移动启动/停止有顿感
- 视角旋转没有加速/减速曲线
- 位置更新与 store 同步可能导致微卡顿

### 问题 3：玩家穿墙 / 碰撞问题
**现状分析：**
- `playerMovement.ts` 中有 `resolveRoomCollision` 函数，使用房间边界 + 门洞检测
- 采用滑动碰撞（先试 X 轴滑动，再试 Z 轴滑动）
- 碰撞检测只考虑房间外墙，不考虑家具等室内障碍物

**具体问题：**
- 没有家具碰撞 → 玩家可以穿过沙发、桌子等
- 角落碰撞可能有问题（滑动逻辑在某些角度可能卡住）
- 玩家半径 0.3m，门洞 padding 0.05m，需要验证是否合理

### 问题 4：小地图显示/交互问题
**现状分析：**
- `Minimap.tsx` 使用 Canvas 2D 渲染，支持拖拽、缩放、重置
- 尺寸固定 200x200px
- 自动适配缩放 `computeFitZoom`，paddingFactor 1.25

**具体问题：**
- 小地图尺寸 200px 偏小，多房间时显示拥挤
- 房间名称可能重叠或显示不全
- 初始视图可能没有完整显示所有房间
- 没有"跟随玩家"模式，拖拽后需要手动重置
- 物体标记（绿色点）没有图例说明

## 实施步骤

### 阶段一：修复物体/家具摆放悬空

**步骤 1.1：审视并修正 RoomDecorations 中所有家具的 Y 位置**
- 逐个房间检查每个家具 group 的 position.y
- 确保 `position.y = size.y / 2`（如果模型中心在几何中心）
- 放在地面上的物体：底部 y=0，中心 y=size.y/2
- 放在家具表面上的物体：底部 y=表面高度，中心 y=表面高度 + size.y/2

**步骤 1.2：统一 FurnitureModel 的对齐方式**
- 确认 FurnitureModel 的 `-visualHeight / 2` 偏移是否正确
- 确保所有家具模型的几何原点在底部，或统一偏移逻辑

**步骤 1.3：验证表面放置的物体**
- 餐桌上的杯子、盘子等是否正确落在桌面上
- 床头柜上的台灯是否正确放置
- 检查 `surfaceHeight` 配置是否正确

### 阶段二：优化玩家移动流畅度

**步骤 2.1：添加位置平滑插值**
- 在 FirstPersonControls 中添加 `smoothedPosition` ref
- 使用 lerp 每帧平滑相机位置到目标位置
- 平滑系数可调（如 0.15-0.25）

**步骤 2.2：优化视角旋转**
- 添加视角旋转的平滑插值
- 鼠标移动时累积角速度，逐渐衰减
- 避免直接 setRotation 的生硬感

**步骤 2.3：优化输入响应**
- 考虑添加输入加速/减速曲线
- WASD 按键的启动和停止更自然

### 阶段三：修复墙体碰撞 & 添加家具碰撞

**步骤 3.1：添加家具 AABB 碰撞**
- 在 `playerMovement.ts` 中添加家具碰撞检测
- 从 task.containers 中提取家具的世界坐标和尺寸
- 实现玩家圆形 vs 家具矩形的碰撞检测 + 滑动响应

**步骤 3.2：优化墙角滑动**
- 检查当前滑动逻辑在斜向移动撞墙时是否平滑
- 修复可能的卡住问题

**步骤 3.3：验证门洞通行**
- 确保玩家能顺利通过门洞
- 调整玩家半径或门洞 padding 确保手感自然

### 阶段四：优化小地图

**步骤 4.1：增大尺寸 & 优化布局**
- 从小地图尺寸从 200px 调整到 240-280px
- 优化控制按钮布局

**步骤 4.2：添加玩家跟随模式**
- 默认开启"相机跟随玩家"模式
- 玩家移动时小地图视图自动跟随
- 拖拽后暂时退出跟随，点击重置按钮恢复

**步骤 4.3：优化显示内容**
- 确保所有任务房间都能完整显示
- 房间名称不重叠
- 添加简单图例（玩家、物体）

**步骤 4.4：优化缩放适配**
- 调整 paddingFactor 确保边界房间不被裁切
- 多房间布局时自动计算最佳缩放

## 潜在风险与注意事项

1. **家具碰撞可能影响路径**：添加家具碰撞后，玩家可能需要绕路，需确保任务可达
2. **移动平滑可能影响操作手感**：插值系数需要调优，太大会延迟，太小没效果
3. **小地图尺寸变化可能影响 HUD 布局**：需要确保与其他 UI 元素不重叠
4. **家具 Y 位置修正可能导致物体穿模**：需要仔细验证每个家具的视觉效果
