# 场景黑屏问题修复计划

## 问题描述
进入游戏后 3D 场景全黑，只有 HUD 显示正常。

## 可能原因分析

### 原因 1：Vite 开发服务器不稳定（最可能）
- 控制台显示大量 `net::ERR_ABORTED` 错误
- Vite 服务器连接断开重连
- 端口号频繁变化（5191→5192→5194）
- 核心模块（react-three-fiber、three、drei）加载失败

### 原因 2：ChaosEffect 光照修改导致
- 修改环境光时可能把强度调到 0 或 NaN
- 但代码中有 lerp 保护，且混乱值 19% 时光照应该接近正常

### 原因 3：Vignette 球体挡住视线
- 球体半径 15，BackSide 材质
- 但透明度很低（~4.5%），且不写深度，不应该挡住场景

## 修复步骤

### 步骤 1：重启开发服务器（验证是否是 Vite 问题）
- 停止当前服务器
- 清理 Vite 缓存
- 重新启动
- 观察是否恢复正常

### 步骤 2：如果重启后仍黑屏，快速回滚高风险改动
- 先禁用 ChaosEffect 中的光照修改逻辑
- 禁用相机抖动
- 禁用 vignette 球体
- 逐步排查是哪个部分导致的

### 步骤 3：定位根因并修复
- 如果是光照修改问题：改为更安全的方式（直接操作 JSX 中的光照组件，而不是运行时查找）
- 如果是相机抖动问题：降低优先级或移除
- 如果是 vignette 问题：调整实现方式

### 步骤 4：验证
- 进入游戏确认场景正常显示
- 确认混乱值效果正常工作
- 运行 build 和测试
