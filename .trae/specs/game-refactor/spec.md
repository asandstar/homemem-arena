# Echo House: Memory Butler - Product Requirement Document

## Overview
- **Summary**: 将 HomeMem Arena v2 从"家务机器人记忆研究平台"改造成一个游戏性更高的 3D 第一人称家务记忆闯关小游戏。玩家扮演记忆模块故障的家务机器人，在会捣乱的房子里限时完成家务任务。
- **Purpose**: 通过游戏化改造，提升用户体验，使原本的研究平台更具娱乐性和可玩性，同时保留核心的记忆研究数据采集能力。
- **Target Users**: 游戏玩家、AI 研究人员、开发者、比赛评委

## Goals
- 将项目改造成名为"Echo House: Memory Butler（记忆宅邸：失忆家务机器人）"的网页小游戏
- 新增有限记忆槽系统（3个槽位，可锁定，可覆盖）
- 新增混乱值系统（Chaos Meter），达到100时关卡失败
- 新增评分和 combo 系统，结算时给出评级和称号
- 将四个任务改造成游戏关卡，每个关卡有独特的捣乱事件和教学内容
- 重构 HUD 为游戏界面风格
- 添加即时游戏反馈效果
- 改造结算页为游戏结算界面
- 保留 AI 分析模块并改造成机器人诊断报告形式

## Non-Goals (Out of Scope)
- 不添加新的物理引擎
- 不添加声音系统
- 不添加移动端触屏控制
- 不引入外部大型 GLTF / FBX / 贴图资产；允许新增程序化低多边形模型组件
- 不改变现有技术栈

## Background & Context
- 现有项目：HomeMem Arena v2，一个基于 React + TypeScript + Three.js 的家务机器人记忆研究平台
- 现有功能：3D 房间系统、物体和容器系统、四个任务配置、事件系统、记忆系统、HUD、结果分析页
- 改造方向：保留技术栈和核心系统，重构体验使其更像游戏

## Functional Requirements

### FR-1: 有限记忆槽系统
- **FR-1.1**: 玩家默认拥有 3 个记忆槽
- **FR-1.2**: 靠近物体或观察到事件后，按 E 键可以保存一条记忆
- **FR-1.3**: 记忆槽满后，需要选择覆盖旧记忆
- **FR-1.4**: 每条记忆包含 objectName、roomName、containerName、state、timestamp
- **FR-1.5**: 支持锁定一条记忆，锁定的记忆不会被覆盖

### FR-2: 混乱值系统 (Chaos Meter)
- **FR-2.1**: 时间流逝会缓慢增加混乱值
- **FR-2.2**: 放错物体会增加混乱值
- **FR-2.3**: 重复搜索同一房间会增加混乱值
- **FR-2.4**: 忘记关闭冰箱或柜子会增加混乱值
- **FR-2.5**: 混乱值越高，脚本事件触发越频繁
- **FR-2.6**: 混乱值达到 100 时关卡失败

### FR-3: 评分和 Combo 系统
- **FR-3.1**: 正确拾取目标物体加分
- **FR-3.2**: 正确放置物体加分
- **FR-3.3**: 使用记忆找到被移动物体加分
- **FR-3.4**: 错误放置扣分
- **FR-3.5**: 重复搜索扣分
- **FR-3.6**: 超时扣分
- **FR-3.7**: 连续正确操作增加 combo
- **FR-3.8**: 错误操作打断 combo
- **FR-3.9**: combo 影响最终得分倍率
- **FR-3.10**: 结算时给出 S/A/B/C/D 评级和称号

### FR-4: 关卡改造
- **FR-4.1**: 关卡 1「出门大作战」- 准备出门任务改造
- **FR-4.2**: 关卡 2「餐桌混乱」- 收拾餐桌任务改造
- **FR-4.3**: 关卡 3「洗衣幽灵」- 洗衣分类任务改造
- **FR-4.4**: 关卡 4「早餐时间循环」- 早餐准备与归位任务改造

### FR-5: HUD 游戏化重构
- **FR-5.1**: 左上角显示关卡名、任务 checklist、剩余时间
- **FR-5.2**: 顶部显示混乱值、进度条、combo
- **FR-5.3**: 右上角显示得分、评级预估、当前持有物体
- **FR-5.4**: 左下角显示操作提示
- **FR-5.5**: 右下角显示小地图（只显示已访问房间）
- **FR-5.6**: 底部中央显示记忆槽 UI

### FR-6: 即时反馈
- **FR-6.1**: 正确放置时出现绿色发光提示
- **FR-6.2**: 错误放置时出现红色警告
- **FR-6.3**: 捣乱事件触发时出现短暂剧情提示
- **FR-6.4**: 保存记忆时记忆槽闪烁
- **FR-6.5**: combo 增加时有浮动文字
- **FR-6.6**: 混乱值升高时 HUD 有轻微故障效果

### FR-7: 结算页改造
- **FR-7.1**: 显示最终得分、评级、称号
- **FR-7.2**: 显示完成时间、combo 最大值、错误放置次数、重复搜索次数、使用记忆次数
- **FR-7.3**: 显示 AI 机器人诊断报告

### FR-8: AI 模块改造
- **FR-8.1**: analyzeSession 重命名为 generateRobotDiagnosis
- **FR-8.2**: updateRobotMemory 重命名为 memorySlotSuggestion
- **FR-8.3**: 结果页显示 AI 诊断报告

### FR-9: 低多边形模型精致化
- **FR-9.1**: 所有关键任务物体必须使用可识别的程序化低多边形模型，不能继续使用纯色方块或色块占位
- **FR-9.2**: 所有模型需要保持统一风格：低多边形、圆角感、暖色家居、轻微神秘感、机器人 HUD 科幻感
- **FR-9.3**: 关键任务物品需要具备清晰剪影，缩小后仍能识别
- **FR-9.4**: 可交互物体需要 hover 高亮、靠近提示、任务目标描边或呼吸光
- **FR-9.5**: 背景家具和任务物体需要有视觉层级，任务物体更醒目，装饰物更低饱和
- **FR-9.6**: 场景中必须开启阴影或接触阴影，物体不能像漂浮在地面上
- **FR-9.7**: 模型必须轻量化，不引入外部大型资产，不引入复杂后处理，不影响浏览器流畅运行

### FR-10: 关键任务物品模型库
请重构或扩展 ObjectGeometries.tsx，至少实现以下模型：
1. KeyModel - 钥匙环、钥匙柄、齿形末端，不能只是黄色小方块
2. PhoneModel - 扁平机身、屏幕、摄像头点、微弱屏幕发光
3. UmbrellaModel - 长柄、弯钩、折叠伞面、伞尖
4. MilkCartonModel - 屋顶形纸盒、正面标签、小盖子
5. CerealBoxModel - 高盒体、正面图案块、顶部封口
6. CupModel - 杯身、把手、杯口内侧
7. BowlModel - 浅碗形状、内外颜色区分
8. PlateModel - 扁圆盘、边缘凸起
9. RemoteModel - 细长机身、按钮阵列
10. ClothModel - 柔软折叠布料形状，带折线，不使用平面方块
11. TowelModel - 长条布料、折痕线、轻微厚度
12. TrashModel - 揉皱纸团或包装袋形状

### FR-11: 家具与房间生活感
请新增或优化 Furniture / Container 模型：
1. FridgeModel - 高柜体、门缝、把手、开门状态、内部层架
2. CabinetModel - 上柜、下柜、门板、把手、开关状态
3. SinkModel - 台面、水槽凹陷、水龙头
4. DishwasherModel - 门板、把手、状态灯
5. SofaModel - 底座、靠背、扶手、靠枕
6. CoffeeTableModel - 桌面、桌腿、少量杂物
7. BedModel - 床架、被子、枕头
8. DeskModel - 桌面、抽屉、台灯
9. LaundryBasketModel - 篮筐、标签牌、内部空间
10. EntranceTrayModel - 浅托盘、目标区域高亮

每个房间至少要有 3 个能表明用途的视觉锚点：
- 玄关：门、鞋柜、托盘、伞架、挂钩、地垫
- 客厅：沙发、茶几、电视柜、地毯、落地灯、靠枕
- 厨房：冰箱、橱柜、水槽、台面、垃圾桶、洗碗机
- 卧室：床、床头柜、书桌、衣柜、台灯、散落衣物
- 洗衣区：洗衣机、三个洗衣篮、毛巾架、洗衣液、散落衣物
- 餐厅：餐桌、椅子、盘子、杯子、餐巾、吊灯

### FR-12: 材质、光照和视觉反馈
- **FR-12.1**: 建立统一 palette：背景家具低饱和暖色，任务物体更高饱和，危险提示红橙，正确目标绿青，记忆相关蓝紫
- **FR-12.2**: 不同材质要有不同 roughness / metalness / emissive，例如屏幕微发光、金属钥匙略反光、布料低高光
- **FR-12.3**: 每个房间有局部灯光，客厅偏暖，厨房略冷，卧室柔和
- **FR-12.4**: hover 时物体有描边或发光
- **FR-12.5**: 正确放置时目标区域出现绿色脉冲光
- **FR-12.6**: 错误放置时物体抖动并显示红色警告
- **FR-12.7**: 捣乱事件发生时出现短暂特效，例如猫脚印、物体滑动轨迹、灯光闪烁
- **FR-12.8**: 保存记忆时有从物体到记忆槽的视觉反馈

## Non-Functional Requirements
- **NFR-1**: 保持与现有技术栈兼容（React + TypeScript + Vite + @react-three/fiber + drei + zustand）
- **NFR-2**: 复用现有系统，减少代码改动
- **NFR-3**: 构建成功且控制台无严重报错
- **NFR-4**: 游戏界面响应流畅，无明显卡顿

## Constraints
- **Technical**: 使用现有技术栈，不引入新依赖
- **Business**: 保留 AI 能力展示，方便比赛说明
- **Dependencies**: 复用现有任务配置、房间系统、物体系统

## Assumptions
- [ ] 用户熟悉第一人称游戏操作
- [ ] 游戏目标是娱乐性和研究性的结合
- [ ] 玩家愿意管理有限的记忆资源

## Acceptance Criteria

### AC-1: 首页游戏化改造
- **Given**: 用户访问首页
- **When**: 页面加载完成
- **Then**: 首页显示游戏标题"Echo House: Memory Butler"，有开始游戏按钮，看起来像游戏首页
- **Verification**: `human-judgment`

### AC-2: 关卡选择页改造
- **Given**: 用户点击开始游戏
- **When**: 进入关卡选择页面
- **Then**: 页面显示四个关卡卡片，每个卡片有关卡名、描述、难度标识，看起来像关卡选择界面
- **Verification**: `human-judgment`

### AC-3: 有限记忆槽系统
- **Given**: 玩家进入游戏
- **When**: 靠近物体按 E 键
- **Then**: 记忆被保存到记忆槽中，记忆槽满时提示选择覆盖，可锁定记忆防止被覆盖
- **Verification**: `human-judgment`

### AC-4: 混乱值系统
- **Given**: 玩家正在游戏中
- **When**: 进行操作或时间流逝
- **Then**: 混乱值随时间和错误操作增加，达到 100 时关卡失败
- **Verification**: `programmatic`

### AC-5: 评分和 Combo 系统
- **Given**: 玩家完成关卡
- **When**: 进入结算页面
- **Then**: 显示最终得分、评级（S/A/B/C/D）、称号、combo 最大值
- **Verification**: `human-judgment`

### AC-6: 第一关完整可玩
- **Given**: 用户选择第一关「出门大作战」
- **When**: 完成所有游戏流程
- **Then**: 可以找到钥匙、手机、雨伞并放到玄关托盘，有至少 2 个捣乱事件，关卡可以正常完成
- **Verification**: `human-judgment`

### AC-7: HUD 游戏化
- **Given**: 玩家进入游戏
- **When**: 查看界面
- **Then**: HUD 显示符合游戏化布局要求（左上角任务、顶部混乱值、右上角得分、底部记忆槽等）
- **Verification**: `human-judgment`

### AC-8: 即时反馈效果
- **Given**: 玩家进行游戏操作
- **When**: 正确放置、错误放置、保存记忆、combo 增加
- **Then**: 出现相应的视觉反馈效果
- **Verification**: `human-judgment`

### AC-9: 游戏化结算页
- **Given**: 玩家完成关卡
- **When**: 进入结算页面
- **Then**: 结算页显示游戏化的统计数据和 AI 诊断报告
- **Verification**: `human-judgment`

### AC-10: 其他关卡可进入
- **Given**: 用户选择其他关卡
- **When**: 点击关卡卡片
- **Then**: 关卡可以正常进入和游玩
- **Verification**: `human-judgment`

### AC-11: 构建成功
- **Given**: 代码修改完成
- **When**: 运行 npm run build
- **Then**: 构建成功，无错误
- **Verification**: `programmatic`

### AC-12: 无严重报错
- **Given**: 游戏运行中
- **When**: 进行各种操作
- **Then**: 控制台无严重报错（红色错误）
- **Verification**: `programmatic`
