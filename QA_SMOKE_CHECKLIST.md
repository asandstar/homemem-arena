# HomeMem Arena QA Smoke Checklist

> 当前验收清单，更新于 2026-07-10。  
> 历史试玩报告不能替代本清单。完整验收门槛见 [产品、游戏与研究设计基线](docs/product-research-game-design.md#9-验收门槛)。

## 1. 自动检查

- [ ] `npm test` 全部通过；
- [ ] `npm run lint` 0 error；
- [ ] `npm run build` 通过，无意外的大包/资源错误；
- [ ] `npm run qa` 在断网环境可运行；
- [ ] 4 个关卡都有最短合法通关集成测试；
- [ ] Session JSON 通过 schema 校验。

## 2. 首页与任务选择

- [ ] `/` 无白屏和 Console error；
- [ ] 首屏能看到“回声屋”、3 槽记忆和主要 CTA；
- [ ] CTA 进入 `/tasks`；
- [ ] 任务页按顺序显示 4 关；
- [ ] 每张卡显示时间、难度、记忆主题和核心挑战；
- [ ] 卡片可用鼠标、Tab + Enter/Space 进入；
- [ ] 未知 task 路由有明确错误或回到任务列表。

## 3. 欢迎与任务简报

- [ ] 首次进入只显示最上层欢迎弹窗，不发生视觉重叠；
- [ ] 欢迎弹窗关闭后显示任务简报；
- [ ] 阅读欢迎/简报时计时不增长；
- [ ] 阅读欢迎/简报时混乱值不增长；
- [ ] 阅读欢迎/简报时脚本事件不触发；
- [ ] 点击“开始任务”后才进入 `playing`；
- [ ] 返回任务列表不会生成失败结果。

## 4. 基础 3D 操作

### 移动与视角

- [ ] `W/S/A/D` 与方向键方向正确；
- [ ] 鼠标左键拖动可以调整 yaw/pitch；
- [ ] `V` 切换第一人称/俯视，方向与位置保持一致；
- [ ] 撞墙沿墙滑动，不穿墙、不持续抖动；
- [ ] 门洞可双向通过，切换后不被弹回；
- [ ] 家具碰撞不会把玩家推出房间。

### 交互

- [ ] `F` 在空手时拾取最近有效物体；
- [ ] `F` 在持有物体时放到最近有效容器；
- [ ] `F` 在没有其他动作时打开/关闭容器；
- [ ] `E` 只保存或更新附近物体记忆；
- [ ] 鼠标点击与 `F` 产生相同的状态、step、event 和目标结果；
- [ ] 失败交互显示具体原因且不会破坏实体状态；
- [ ] 每个离散命令只记录一次。

## 5. HUD 与反馈

- [ ] 当前目标、持有物、得分、Combo、混乱值、记忆槽可见；
- [ ] HUD 只突出一个当前可执行目标，目标推进后正确切换；
- [ ] 无目标进展 20 秒出现一级提示，45 秒出现二级策略提示；
- [ ] 目标推进后停滞提示立即清除，下一段停滞重新计时；
- [ ] 每次提示只记录一个 `flow_intervention`，包含 level、goalId 和 progressStallMs；
- [ ] `Tab`、`R`、`H`/`Esc` 可以收起并恢复对应面板；
- [ ] 小地图房间、门和玩家方向正确；
- [ ] 正确放置有视觉、音效、分数和目标反馈；
- [ ] 错误放置显示原因、打断 Combo、增加相应混乱；
- [ ] 记忆过期不只靠颜色提示；
- [ ] Console 不按帧刷相同 warning；
- [ ] HUD 在 1280px 和 768px 宽度不遮挡核心操作。
- [ ] 结果指标区展示辅助触发次数、最长目标间隔和操作成功率，并标注为过程指标而非主观心流结论。

## 6. 时间、混乱与失败

- [ ] 每关使用自己的 `timeLimit`；
- [ ] 剩余时间只在 `playing` 阶段变化；
- [ ] 到时触发一次明确的 timeout 失败；
- [ ] 混乱增长速率与帧率无关；
- [ ] 混乱达到阈值时反馈逐级增强；
- [ ] 混乱 100 触发一次失败并停止游戏输入；
- [ ] 失败后可以重试，Session 状态为 `failed`；
- [ ] 主动返回和失败能够区分为 `aborted` / `failed`。

## 7. 脚本事件

- [ ] 相同 task version + seed + command 序列产生相同事件；
- [ ] 每个一次性事件只触发一次；
- [ ] 事件不因高帧率提前触发；
- [ ] 移动物体时同步更新 `currentRoom`、`position`、`status`、`placedIn` 和容器成员关系；
- [ ] 相关记忆被标记过期；
- [ ] 事件的视觉、声音、Toast 与真实状态变化一致；
- [ ] Session 记录事件条件、时间、step、old/new state 和 affected entities。

## 8. 四关通关

### 餐桌混乱

- [ ] 初始“干净杯保持自由”不会重复奖励；
- [ ] 脏杯、盘子、纸巾、遥控器都能正确归位；
- [ ] 餐盘精回置物体后，已完成目标按设计决定保持里程碑或重新要求处理；
- [ ] 所有终局条件满足后进入 Probe。

### 出门大作战

- [ ] 客厅、卧室、玄关、厨房连通；
- [ ] 抽屉中的手机需打开容器后获取；
- [ ] 钥匙移动后旧记忆过期，可重新验证；
- [ ] 3 件物品归位后进入 Probe。

### 洗衣幽灵

- [ ] 8 件对象初始可恢复地存在；
- [ ] 3 类衣物只能进入正确篮子；
- [ ] 扰动后数量和位置状态一致；
- [ ] 全部分类后进入 Probe。

### 早餐时间循环

- [ ] 阶段顺序为准备 → 确认使用 → 归位 → 关闭；
- [ ] 已完成的阶段 milestone 不会因后续归位撤销；
- [ ] 终局检查只要求最终应保持的状态；
- [ ] 牛奶时间压力和容器事件实际改变对应规则；
- [ ] 合法流程能够完成且进入 Probe。

## 9. Probe、结果与重玩

- [ ] 关卡完成/失败后先进入 `/probe/:taskId`；
- [ ] Probe 期间场景状态冻结；
- [ ] 题目明确询问初始、事件后或最终状态；
- [ ] 每题记录答案、正确答案、反应时间、记忆类型和关联对象/事件；
- [ ] 最后一题后只 finalize 一次；
- [ ] 结果页显示任务和 Probe 两类指标；
- [ ] 诊断结论可追溯到 Session 证据；
- [ ] “再玩一次”进入正确 `/play/:taskId`；
- [ ] 直接访问无效结果 URL 会恢复或重定向，不会永久显示“加载中”。

## 10. 研究数据完整性

- [ ] 导出包含 schema/app/task/scene/analysis version；
- [ ] 包含 seed 和 condition；
- [ ] commands、actions、steps 数量关系可解释；
- [ ] pose trace 已降采样且时间戳单调；
- [ ] observation、visible objects、object/container delta 非占位空数组；
- [ ] 记忆槽写入/更新/过期与 Session memory events 一致；
- [ ] scripted events 与游戏中实际发生的事件一致；
- [ ] Probe 已包含在 outcome metrics 中；
- [ ] 原始数据和派生指标分区；
- [ ] JSON 预览对特殊字符安全，不执行 HTML。

## 每次提交最小 Smoke

- [ ] 自动测试、lint、build 全绿；
- [ ] 第一关从简报完整走到 Probe 和结果；
- [ ] 简报期间完全暂停；
- [ ] 键盘 F/E 的事件记录正确；
- [ ] 一个扰动能使记忆过期并被重新验证；
- [ ] 重玩路由正确；
- [ ] 导出 Session 通过 schema；
- [ ] Console 无 error 和高频重复 warning。
