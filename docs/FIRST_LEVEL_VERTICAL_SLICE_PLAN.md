## 一、玩法重构

### 当前问题

* 玩家可能不理解为什么必须使用记忆系统
* 猫事件的惊喜感不足
* 记忆过期后的引导不够清晰

### 设计方案

#### 1. 唯一核心目标

**「找到钥匙并放到玄关托盘」**

* 第一关只保留钥匙作为任务目标（已实现）
* 手机、雨伞转为背景装饰（已实现）
* 聚焦记忆循环体验

#### 2. 必须体验的记忆循环

```
发现钥匙 → 保存记忆 → 猫推走钥匙 → 记忆过期提示 → 重新寻找 → 更新记忆 → 完成任务
```

#### 3. 猫事件触发逻辑

**基于玩家行为触发**（已实现）

* 触发条件：玩家离开客厅区域（`currentRoom !== 'living'`）
* 目的：制造"你不在场时发生了变化"的惊喜感
* 事件表现：钥匙从茶几移动到沙发旁（坐标：`{ x: 1.5, y: 0, z: -1.5 }`）

#### 4. 玩家为什么必须使用 E 保存记忆

* **第一次靠近钥匙时**：弹窗提示「🧠 第一次靠近钥匙！按 E 保存位置记忆，捣乱事件后可以回顾」（已实现）
* **猫事件后**：如果没有保存记忆，玩家需要重新搜索整个房间
* **奖励机制**：保存记忆 +50 分，更新记忆 +30 分（已实现）

#### 5. 旧记忆如何失效

* 猫事件触发时，自动标记钥匙的旧记忆为「过期」（已实现）
* 过期记忆在记忆槽中显示红色闪烁 + 抖动动画（已实现）
* 屏幕边缘红色脉动效果提示过期记忆存在（已实现）

#### 6. 玩家如何理解"更新记忆"

* 靠近过期记忆对应的物品时，显示「⚠️ 记忆已过期！按 E 更新」（已实现）
* 更新记忆后，记忆槽恢复正常颜色
* 更新时播放特效和音效

## 二、第一关流程设计

### 时间线

#### 0-10 秒：出生与观察

| 维度 | 内容 |
| --- | --- |
| 玩家看到 | 客厅全景，沙发、茶几、电视等家具，茶几上的金色钥匙 |
| 玩家知道 | 任务简报刚结束，目标是找到钥匙放到玄关托盘 |
| 系统反馈 | 小地图显示房间布局，任务目标高亮，底部记忆槽显示 |
| 可能困惑 | 不知道钥匙在哪里 |
| 改进方案 | 钥匙有微弱金色发光，茶几位置在视野中心 |

#### 10-30 秒：第一次交互

| 维度 | 内容 |
| --- | --- |
| 玩家目标 | 探索房间，找到钥匙 |
| 玩家操作 | WASD移动，鼠标转视角 |
| 系统反馈 | 靠近物品时显示名称标签，距离提示 |
| 可能困惑 | 不知道怎么互动 |
| 改进方案 | 右下角显示操作提示，靠近钥匙时出现拾取提示 |

#### 30-60 秒：第一次保存记忆

| 维度 | 内容 |
| --- | --- |
| 玩家目标 | 找到钥匙并保存记忆 |
| 玩家操作 | 靠近钥匙 → 按 E 保存记忆 |
| 系统反馈 | 记忆槽动画，分数+50，音效提示，弹窗解释"记忆已保存！环境变化后可以回顾" |
| 可能困惑 | 为什么要保存记忆？ |
| 改进方案 | 保存后弹出解释，建立记忆价值认知 |

#### 60-120 秒：环境变化

| 维度 | 内容 |
| --- | --- |
| 玩家目标 | 离开客厅去玄关查看 |
| 玩家操作 | 走向玄关方向，穿过门口 |
| 系统反馈 | 离开客厅时触发猫事件，屏幕提示"🐱 啪嗒——钥匙猫把钥匙扒拉走了！"，钥匙记忆标记为过期，记忆槽红色闪烁，屏幕红色脉动 |
| 可能困惑 | 发生了什么？为什么记忆失效了？ |
| 改进方案 | 事件动画展示，明确说明猫推走了钥匙 |

#### 120-180 秒：重新寻找和更新记忆

| 维度 | 内容 |
| --- | --- |
| 玩家目标 | 找到钥匙新位置，更新记忆 |
| 玩家操作 | 返回客厅 → 搜索沙发区域 → 发现钥匙 → 按 E 更新记忆 |
| 系统反馈 | 钥匙在沙发旁，靠近时显示"⚠️ 记忆已过期！按 E 更新"，更新后记忆恢复正常，分数+30 |
| 可能困惑 | 不知道钥匙去哪里了 |
| 改进方案 | 猫脚印引导，过期记忆显示旧位置提示 |

#### 180-300 秒：完成和结算

| 维度 | 内容 |
| --- | --- |
| 玩家目标 | 把钥匙放到玄关托盘 |
| 玩家操作 | 拾取钥匙 → 走到玄关 → 放到托盘 |
| 系统反馈 | 放置成功动画，任务完成提示，结算界面显示记忆表现 |
| 可能困惑 | 不知道托盘在哪里 |
| 改进方案 | 托盘高亮，小地图标记目标位置 |

* 场景像测试房间，缺乏生活感
* 关键物品不够突出

**玩家第一眼应该知道"哪里重要"**

### 关键视线

* 出生点 → 茶几（钥匙）：直线视线，无遮挡
* 茶几 → 玄关门口：通过门口，有明确路径
* 沙发 → 茶几：猫事件后需要回头搜索

### 玩家移动路径

```
出生点 → 茶几（存记忆）→ 玄关门口（触发猫事件）→ 返回客厅 → 沙发旁（找钥匙/更新记忆）→ 玄关（放钥匙）
```

### 突出与隐藏

* **突出**：钥匙（金色发光+脉冲）、托盘（黄色高亮）、门口（光线效果）
* **隐藏**：猫事件后钥匙在沙发旁（需要搜索）

## 六、视觉风格要求

### 当前问题

* 纯色大块墙面，缺乏质感
* 家具颜色过于单调
* 测试场感觉明显

### 目标风格

**Cozy Stylized Indie Game**

### 设计要求

1. **减少纯色大块**
   * 墙面添加轻微纹理（使用噪点材质）
   * 地板添加木纹（使用程序化纹理）
   * 家具添加材质细节（粗糙度变化）

2. **增加生活感**
   * 沙发上的抱枕（已实现：红、青、黄三色）
   * 茶几上的杂志/果盘（待添加）
   * 书架上的书籍（已实现：BookshelfFallback）
   * 玄关的鞋子（已实现：ShoesFallback）

3. **关键物体高亮**
   * 钥匙：金色发光，脉冲动画（已实现）
   * 托盘：黄色边框，目标标识（已实现）
   * 门口：光线效果（待添加）

4. **家具低饱和**
   * 沙发：深棕色（#8b5a2b）
   * 茶几：原木色（#8b7355）
   * 墙壁：暖米色（#fff0c8）

5. **任务物体高识别度**
   * 钥匙颜色与环境形成对比（金色 vs 暖色环境）
   * 拾取后有明显的手持效果（已实现）
   * 放置成功有庆祝动画（已实现）

6. **不依赖大量文字说明**
   * 用视觉提示代替文字
   * 动画和音效传达信息

7. **不新增大型模型资产**
   * 使用现有几何体构建
   * 通过材质和颜色区分

## 八、结算设计

### 当前实现

结算界面已包含：
* 结果状态（成功/失败）
* 得分和评级
* 记忆表现统计（保存次数、有效记忆、过期记忆、更新次数）
* 个性化反馈（"你的机器人记忆策略"）

### 反馈文案逻辑

| 条件 | 反馈文案 |
| --- | --- |
| 保存≥2 且 更新≥1 | "记忆大师！你完美地保存并更新了记忆，简直是记忆系统的最佳使用者！" |
| 保存≥1 且 更新≥1 | "反应迅速！猫事件后你很快找到了钥匙并更新了记忆，效率很高！" |
| 保存≥1 | "做得不错！你使用了记忆系统保存位置，下次试试更新记忆吧！" |
| 未使用记忆 | "记忆新手！这次你没有使用记忆系统，但仍然完成了任务。试试保存记忆，会更轻松！" |
| 失败 | "时间到了！下次记得更快一点找到钥匙哦！" |

### 可改进点

* 添加记忆效率百分比（有效记忆 / 总保存次数）
* 添加时间评级（完成时间对应的星级）
* 添加猫事件应对速度评级
