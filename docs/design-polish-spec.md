# HomeMem Arena 视觉与交互规格

> 状态：当前 3D 版本规格  
> 最近更新：2026-07-10  
> 本文只规定玩家可见体验。游戏规则和研究契约见 [产品、游戏与研究设计基线](product-research-game-design.md)。

## 1. 体验目标

视觉与交互服务于三个结果：

1. 玩家快速理解目标、可交互对象和当前风险；
2. 3D 场景保持温暖、奇幻和可读，不用装饰遮盖关键信息；
3. 研究相关的记忆、扰动和结果信息可信、可解释、不伪装成神秘黑箱。

整体风格：**家庭奇幻场景 + 轻量机器人系统界面**。

- 叙事层：紫色、暖黄、柔和渐变、便签和小妖符号；
- 系统层：深色半透明 HUD、青色/紫色记忆信息、清晰的数值反馈；
- 结果层：游戏成就感在前，研究证据在后。

## 2. Design Token

颜色以 `src/index.css` 中的 Tailwind `@theme` 为真值源。

| 语义 | 建议颜色 | 用途 |
|---|---|---|
| Primary | Indigo / Purple | CTA、当前选择、品牌 |
| Success | Emerald | 正确放置、目标完成 |
| Warning | Amber | 记忆过期、时间压力 |
| Danger | Red | 错误、失败、混乱临界 |
| Spatial | Sky blue | 空间记忆 |
| Object | Cyan | 物体状态记忆 |
| Temporal | Purple | 时序记忆 |
| Procedural | Green | 程序记忆 |
| Game surface | Near-black slate | HUD、结果页、日志 |

通用规范：

- 圆角：按钮 8–10px，卡片 12–16px，主要弹窗 16–20px；
- 间距：4 / 8 / 12 / 16 / 24 / 32 / 48；
- 动效时长：即时反馈 120–200ms，面板 200–300ms，剧情事件不超过 5s；
- 正文最小 14px，辅助 HUD 不低于 11px；
- 所有图标按钮必须有 accessible name 和明显 focus 样式。

## 3. 页面规格

### 3.1 首页 `/`

目标：让玩家先感受到“这是一个游戏”，再理解“记忆是核心机制”。

必须包含：

- “回声屋 / 家政小精灵的一天”主标题；
- 3 槽记忆、小妖扰动、Combo、星级四个核心卖点；
- 一句研究层说明，避免首页看起来与 HomeMem 名称无关；
- 唯一主要 CTA“开始实习”；
- 四只小妖和一天四关的简短预告。

桌面首屏应完整露出主要 CTA；移动端允许内容纵向滚动。

### 3.2 任务选择 `/tasks`

目标：让玩家理解顺序、难度和本关记忆主题。

- 固定 4 张关卡卡片；
- 显示时间、编号、难度、记忆类型和一句核心挑战；
- 卡片必须使用 `<button>` 或 `<a>`，支持 Tab、Enter、Space；
- 推荐按故事顺序排列：餐桌混乱 → 出门大作战 → 洗衣幽灵 → 早餐时间循环；
- 未做关卡解锁前，不用视觉暗示存在锁定状态。

### 3.3 3D 游戏 `/play/:taskId`

#### 开始阶段

欢迎弹窗与任务简报只能有一个处于最上层。两者打开时：

- 场景可以预渲染；
- 计时、混乱、事件和输入必须暂停；
- 点击“开始任务”后才进入 `playing`；
- 返回任务列表不产生失败 Session。

#### 3D 可读性

- 当前房间提供清楚的照明和名称；
- 关键物体通过轮廓、材质、颜色或轻量提示与装饰区分；
- hover/nearby/selected/held 四种状态不能只靠同一种颜色表达；
- 容器开合和是否可接收当前物体应有即时反馈；
- 门洞方向和目标房间名称在接近时可见；
- fallback 模型必须仍能明确表达类别。

#### HUD 信息层级

| 优先级 | 内容 |
|---|---|
| P0 | 当前目标、持有物、交互提示、剩余时间/混乱临界 |
| P1 | 记忆槽、分数、Combo、小地图、事件 Toast |
| P2 | 事件日志、详细控制帮助、调试信息 |

HUD 要求：

- 任务目标、记忆槽、小地图和帮助都可收起；
- 目标完成显示稳定的历史状态，不因后续阶段撤销；
- 过期记忆使用图标 + 文案 + 颜色，不只变红；
- 混乱值在 60/80/100 阈值有逐级反馈，避免低值持续闪烁；
- Toast 不遮挡准星、核心目标或弹窗 CTA；
- `H` 隐藏辅助 HUD 后，`Esc` 或可见按钮可以恢复。

#### 控制提示

- `E` 始终表示保存/更新记忆；
- `F` 始终表示世界交互；
- 附近同时存在对象和容器时，要显示本次 F 会执行什么；
- 键盘和鼠标操作必须呈现相同结果和反馈；
- 后续触屏模式应提供虚拟移动和上下文交互按钮。

### 3.4 Memory Probe `/probe/:taskId`

- 顶部显示进度和本轮任务名；
- 每题计时从题目稳定可见后开始；
- 选项按钮可键盘操作；
- 提交后短暂显示反馈，再进入下一题；
- 明确区分“最初”“扰动后”“最终”状态；
- 演示自动答题只在开发/演示模式出现，研究条件下必须关闭；
- Probe 期间不再改变游戏场景。

### 3.5 结果页 `/result/:sessionId`（目标路由）

信息顺序：

1. 完成/失败、星级、称号；
2. 得分、时间、Combo、混乱；
3. 记忆行为与 Probe；
4. 失败证据和可执行策略；
5. 重玩、任务列表、查看研究数据。

结果页不得把“0 次使用记忆”显示为 100% 有效记忆。诊断中的数值必须能从当前 Session 追溯。

### 3.6 研究数据 `/data/:sessionId`

- 顶部显示 schema/app/task/analysis 版本和 seed；
- 将原始事件、派生指标、诊断分区展示；
- JSON 预览必须进行 HTML 转义；
- 提供复制、下载和 schema 校验状态；
- 正式众包环境不直接暴露匿名参与者以外的标识。

## 4. 反馈与动效

| 场景 | 反馈 |
|---|---|
| 拾取 | 对象进入手持位、短音效、动作 Toast |
| 正确放置 | 落位动画、得分浮字、目标/Combo 更新 |
| 错误放置 | 对象轻微抖动、原因文字、混乱变化 |
| 保存记忆 | 槽位写入动画、对象/房间/状态明确显示 |
| 记忆过期 | 槽位失效图标、环境事件线索、可重新验证提示 |
| 小妖扰动 | 角色特效 + 状态变化，二者时间一致 |
| 失败 | 清楚原因、停止输入、重试 CTA |
| 完成 | 短叙事收束后进入 Probe，不直接跳结果 |

所有强闪烁和抖动应支持 `prefers-reduced-motion` 降级。

## 5. 响应式与性能

| 范围 | 策略 |
|---|---|
| ≥1280px | 完整 HUD，面板分布在四角 |
| 768–1279px | 紧凑 HUD，默认收起日志和控制帮助 |
| <768px | 当前仅保证浏览和结果页；完整游戏需触屏控制后再宣称支持 |

性能体验要求：

- 首页和任务页不应加载完整 3D 游戏包；
- 桌面目标 60 FPS，低性能模式至少 30 FPS；
- 高混乱视觉效果不能持续触发全页面重绘；
- 控制台不得按帧刷重复警告；
- 提供关闭阴影/降低特效的质量档位。

## 6. 可访问性

- 所有可点击卡片和图标按钮支持键盘和可见焦点；
- 不用颜色作为唯一状态编码；
- HUD 文字与背景满足合理对比度；
- Toast 使用适当 live region，但避免高频 observation 播报；
- 结果和 Probe 必须能脱离 Canvas 完成阅读与操作；
- 为无法使用第一人称控制的用户保留未来的替代控制模式。

## 7. 视觉验收优先级

### P0

1. 弹窗期间暂停且层级正确；
2. 目标/交互/记忆状态清晰；
3. 完成后进入 Probe；
4. 失败原因和重试明确；
5. 任务卡和主要按钮可键盘操作。

### P1

1. 小妖事件的视觉、声音、状态变化一致；
2. 结果页补齐记忆和 Probe 证据；
3. 研究数据页版本与校验信息；
4. 平板紧凑 HUD；
5. reduced-motion 和低性能模式。

### P2

1. 触屏/手柄控制；
2. 关卡完成后的连续故事过场；
3. 更丰富但不影响识别的角色和环境资产。
