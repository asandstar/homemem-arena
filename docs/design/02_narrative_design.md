# 记忆宅邸：失忆管家 - 叙事设计

> 状态：当前叙事基线  
> 最近更新：2026-07-14  
> 叙事必须服务于可玩性与研究设计；系统规则以 [产品、游戏与研究设计基线](docs/product-research-game-design.md) 为准。

## 1. 名称与核心设定

- **产品名**：记忆宅邸：失忆管家
- **玩家世界名**：记忆宅邸
- **英文叙事名**：Memory Butler
- **主角**：记忆模块出了故障的家务机器人 **MEM-07**
- **研究代号**：受控家务机器人代理 MEM-07

MEM-07 既是面向玩家的角色化表达，也是研究数据和系统诊断中的代理名称。

MEM-07 来到记忆宅邸，需要完成一天的家务任务。但它的记忆模块出了故障，一次只能稳定记住 3 件物品的位置。宅邸里的四个 NPC（钥匙猫、餐盘精、袜子幽灵、早餐闹钟）会移动物品、混淆状态或打乱流程。玩家需要观察、规划、保存记忆、发现过期信息并重新验证，才能在混乱失控前完成家务。

## 2. 叙事与研究映射

| 玩家叙事 | 游戏机制 | 研究含义 |
|---|---|---|
| MEM-07 的 3 格记忆槽 | 3 个可保存/锁定/覆盖的记忆槽 | 有限工作记忆容量 |
| NPC 捣乱事件 | 脚本化环境事件 | 可控环境扰动 |
| 记忆过期/失效 | 记忆槽 outdated、confidence 下降 | 陈旧记忆与状态更新 |
| 混乱值 | 时间、错误和未处理变化的压力 | 任务负荷的游戏化代理，不直接等同科研指标 |
| 关卡后的回忆问答 | Memory Probe | 记忆准确率和反应时间 |
| MEM-07 诊断 | 规则式策略分析 | 从轨迹推断失败模式和策略 |

叙事可以隐藏术语，但不能隐藏规则。玩家必须知道：

- 只能记 3 条；
- 环境可能变化；
- 旧记忆可能不再可靠；
- E 保存/更新记忆，F 与世界交互；
- 关卡结束后还会测试记忆。

## 3. 角色

### 3.1 MEM-07（玩家）

**玩家层人格**：

- 冷静、数据化，略带幽默；
- 会把复杂问题拆成"一件一件来"；
- 偶尔自我鼓励，不长篇说教；
- 失败时承认问题并给出下一次可执行策略；
- 说话使用技术术语，但也会给出人性化的鼓励。

**系统层人格**：

- 数据驱动、简洁、略带冷幽默；
- 不假装真正理解玩家情绪；
- 诊断必须引用真实指标；
- 规则式分析不声称是远程大模型推理。

### 3.2 四只 NPC

| 角色 | 关卡 | 性格 | 扰动机制 | 记忆主题 |
|---|---|---|---|---|
| 钥匙猫 | 出门大作战 | 调皮、爱恶作剧、好奇心强 | 跨房间寻找、位置变化 | 空间记忆 |
| 餐盘精 | 餐桌混乱 | 得意、爱捣乱、不服输 | 相似物体、脏净混淆、回置 | 物体状态 |
| 袜子幽灵 | 洗衣幽灵 | 害羞、爱藏东西、有点胆小 | 衣物移动、额外对象、重新计数 | 空间与计数 |
| 早餐闹钟 | 早餐时间循环 | 强迫症、固执、机械 | 多阶段流程、超时提醒、容器变化 | 程序与时序 |

NPC 的事件不是纯随机惩罚。每次扰动都应：

1. 与本关记忆主题相关；
2. 给出可感知线索；
3. 留下恢复窗口；
4. 迫使玩家重新观察或更新记忆；
5. 在 Session 中记录实际状态变化。

### 3.3 记忆宅邸（旁白）

- **身份**：宅邸本身的叙述者
- **作用**：营造氛围、描述场景变化、推进剧情
- **对话风格**：诗意、神秘、略带悬疑

## 4. 一天的叙事弧

### 关卡 1：初次整理（教学关卡）

- **时间**：第一天
- **主题**：学习基本操作——移动、拾取、放置、保存记忆
- **玩家动机**：从简单的整理任务开始，学习 MEM-07 的基本操作
- **学习目标**：拾取、放置、目标判断、保存第一条记忆
- **冲突**：无捣乱事件，纯教学
- **完成情绪**：MEM-07 发现自己可以靠观察和记忆完成任务

**开场语气示例**：

> MEM-07：「你好，我是 MEM-07。我的记忆模块出了故障，只能同时记住 3 件物品的位置。」

**完成语气示例**：

> MEM-07：「基础操作已记录。接下来的挑战会更有趣——也会更难。」

### 关卡 2：出门大作战

- **时间**：🌅 清晨 08:00
- **主题**：跨房间空间记忆和过期记忆
- **玩家动机**：在主人出门前找齐钥匙、手机和雨伞
- **学习目标**：规划路线、锁定关键位置、发现位置变化后更新记忆
- **冲突**：钥匙猫会推动钥匙，手机声音提供方向但不直接给答案
- **完成情绪**：从"到处找"转变为有计划地搜索

**事件语气示例**：

> 🐱 啪嗒——钥匙滚到了地毯边。刚才的记忆可能已经过期。

### 关卡 3：餐桌混乱

- **时间**：🍽️ 上午 09:30
- **主题**：第一次理解"状态比外形更重要"
- **玩家动机**：把早餐后的餐桌恢复整洁
- **学习目标**：拾取、放置、目标判断、保存第一条记忆
- **冲突**：餐盘精让相似杯子难以区分，并可能把已知物品放回
- **完成情绪**：MEM-07 发现自己可以靠观察和分类压住混乱

**开场语气示例**：

> 餐盘精好调皮……先看清颜色和状态，一件一件分类，我可以的。

**完成语气示例**：

> 餐桌恢复整洁。MEM-07 学会了：长得像，不代表状态一样。

### 关卡 4：洗衣幽灵

- **时间**：👕 下午 14:00
- **主题**：多对象追踪、分类与重新计数
- **玩家动机**：在袜子幽灵的捉迷藏中完成衣物分类
- **学习目标**：维护类别和数量信息、在扰动后重新核对
- **冲突**：衣物移动、篮子线索变化、额外对象出现
- **完成情绪**：玩家能在更高信息负荷下重新建立秩序

**事件语气示例**：

> 👻 一道影子从篮子旁掠过。有件衣物的位置变了——最好重新数一次。

### 关卡 5：早餐时间循环

- **时间**：⏰ 深夜 23:00
- **主题**：完整长程流程记忆
- **玩家动机**：帮助早餐闹钟完成一次正确流程，结束时间循环
- **阶段**：准备食材 → 摆上餐桌 → 确认使用 → 分类归位 → 关闭容器
- **学习目标**：把空间、物体、时序和程序记忆联合使用
- **冲突**：麦片移动、容器自动关闭、牛奶在外时间压力
- **完成情绪**：四类记忆形成完整链路，一天的故事闭合

**完成语气示例**：

> 闹钟终于停了。不是因为每一步都没变化，而是 MEM-07 记住了变化之后该怎么继续。

## 5. 对话系统

### 5.1 对话触发机制

对话通过以下方式触发：

| 触发类型 | 触发条件 | 示例 |
|---------|---------|------|
| `start` | 关卡开始 | `start_task-clean-table` |
| `roomEnter` | 进入房间 | `roomEnter_living` |
| `event` | 特定事件触发 | `memory_save`、`rank_s` |
| `goalComplete` | 目标达成 | `goalComplete_g-dirty-cup` |

### 5.2 对话优先级

| 优先级 | 类型 |
|--------|------|
| 15-20 | 关卡完成、评级对话 |
| 10 | 关卡开场、关键事件 |
| 8 | 目标完成反馈 |
| 6-7 | 记忆保存/过期 |
| 5 | 进入房间 |
| 4 | 放错位置提示 |

### 5.3 对话选项效果

| 效果类型 | 描述 | 值 |
|---------|------|------|
| `score` | 增加分数 | 数值 |
| `hint` | 显示提示 | 提示文本 |

## 6. 文案分层

### 6.1 Briefing：角色化任务动机

用途：让玩家在 10–20 秒内理解"为什么做、做什么、哪里可能变化"。

结构：

1. 一句场景和角色；
2. 2–4 条明确目标；
3. 一条本关扰动预警；
4. 时间限制；
5. 一句 MEM-07 自我策略。

Briefing 打开期间游戏必须暂停。不要在这里堆叠科研术语。

### 6.2 HUD：短、行动导向

- Toast 最多 1–2 句；
- 先说发生了什么，再暗示可采取的行动；
- 不直接泄露隐藏对象的精确位置；
- 同一事件的视觉、声音和文字应表达同一信息；
- 错误反馈必须说明原因，如"容器不接受深色衣物"。

### 6.3 Probe：中性、无叙事诱导

Probe 题目用于研究测量，应避免 NPC 台词、情绪暗示或答案线索。题目必须说明是在问"初始状态""事件后状态"还是"最终状态"。

例如：

- 好：`钥匙最初位于哪个房间？`
- 不好：`钥匙猫最喜欢在哪个房间推钥匙？`

### 6.4 结果页：成就感 + 证据

玩家层先展示任务结果、星级、得分和一句角色反馈；研究层再展示：

- 哪类记忆表现好；
- 哪个扰动后恢复较慢；
- 哪些行为导致重复搜索或错误；
- 下一次可以尝试的具体策略。

不要用没有数据支持的泛泛 AI 夸奖。

## 7. 语气规范

玩家层 MEM-07 文案：

- 冷静、数据化，适合科幻家庭；
- 句子简短，避免连续大段独白；
- 可以使用少量 emoji，但每条最多 1 个主视觉符号；
- 不幼稚化失败，不嘲讽玩家；
- 使用技术术语，但也会给出人性化的鼓励。

研究层 MEM-07 文案：

- 使用明确指标和记忆类型；
- 冷幽默只能作为结尾，不替代建议；
- 避免"准确率提升 12%"等没有计算来源的数字；
- 避免把混乱值直接解释为真实认知负荷。

推荐格式：

```text
MEM-07 诊断：本轮完成 5/5 个目标。
钥匙移动后用了 18 秒完成重新验证，并发生 1 次无效回访。
下次策略：为跨房间关键物体锁定记忆；收到位置变化线索后优先更新。
```

NPC 角色语气规范：

| 角色 | 语气特点 |
|------|---------|
| 钥匙猫 | 用"喵~"结尾，喜欢挑战玩家，调皮 |
| 餐盘精 | 笑声多，喜欢看玩家手忙脚乱，得意 |
| 袜子幽灵 | 说话断断续续，喜欢躲躲闪闪，害羞 |
| 早餐闹钟 | "嘀嘀嘀！"开头，强调流程正确性，机械 |

## 8. 叙事连续性规则

- 任务选择页顺序、时间标签、关卡编号和故事时间必须一致；
- 同一 NPC 角色在任务卡、Briefing、Toast、完成文案中使用同一名称；
- 四关完成能力递进：状态辨别 → 空间更新 → 多对象追踪 → 完整流程；
- 重玩不是剧情倒退，而是"新的模拟轮次"，可使用不同 seed；
- 每关结束都有 NPC 预告下一关内容，形成连贯叙事链条。

## 9. 文案与代码映射

| 文案 | 字段/位置 |
|---|---|
| 任务卡标题与描述 | `src/data/tasks/*.ts` 的 `name` / `description` |
| 任务简报 | `briefing` |
| 玩家策略 | `systemPrompt`（后续建议改名 `playerThought`） |
| 完成/失败文案 | `completionText` / `failureText` |
| NPC 事件 | `scriptedEvents[].message` |
| 事件类别 | `toastType` / `eventEffect` |
| 对话系统 | `src/dialog/dialogs.ts` |
| 结果诊断 | `src/pages/ResultPage.tsx` 与分析模块 |
| Probe | `probes[]`，应保持中性 |

## 10. 叙事验收清单

- [ ] 第一关（教学）20 秒内能理解目标和 3 槽限制；
- [ ] 四个 NPC 各自对应明确且不同的记忆挑战；
- [ ] 所有扰动都提供线索和恢复方式；
- [ ] 玩家层始终以 MEM-07 为主角；
- [ ] MEM-07 在研究/系统语境出现并有映射说明；
- [ ] Probe 不泄露答案、不混淆初始/最终状态；
- [ ] 结果诊断的每条结论都有 Session 证据；
- [ ] 完成和失败文案都给出下一步意义，不使用虚构指标；
- [ ] 对话系统能触发场景氛围描述和 NPC 个性对话；
- [ ] 关卡间叙事连贯，每关结束预告下一关内容。
