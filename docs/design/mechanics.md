# HomeMem Arena - 核心机制

## 1. 记忆系统

### 1.1 记忆槽

玩家拥有 3 个记忆槽，每个记忆槽可以保存一件物品的位置信息。

```
记忆槽状态：
├── 空槽 → 可保存新记忆
├── 已保存 → 显示物品名称和位置
├── 已过期 → 显示红色警告，记忆可能不准确
└── 已锁定 → 记忆不会过期
```

### 1.2 记忆类型

| 类型 | 图标 | 描述 |
|------|------|------|
| 空间记忆 | 📍 | 记录物品所在房间和位置 |
| 物体记忆 | 📦 | 记录物品的属性和类别 |
| 时间记忆 | ⏱️ | 记录物品被移动的时间 |
| 程序记忆 | 🔄 | 记录操作序列和流程 |

### 1.3 记忆操作

| 操作 | 按键 | 描述 |
|------|------|------|
| 保存记忆 | E | 保存附近物品的记忆 |
| 更新记忆 | E | 更新已保存物品的记忆 |
| 锁定记忆 | 点击锁图标 | 锁定记忆防止过期 |
| 删除记忆 | 点击垃圾桶 | 删除记忆释放槽位 |

### 1.4 记忆过期机制

- 记忆有置信度值（0-100%）
- 置信度随时间下降
- 混乱值越高，置信度下降越快
- 置信度低于阈值时标记为过期
- 过期记忆可能包含错误信息

## 2. 混乱值系统

### 2.1 混乱值定义

混乱值（0-100%）代表系统的失控程度。

### 2.2 混乱值影响

| 混乱值范围 | 效果 |
|-----------|------|
| 0-30% | 正常状态，物品稳定 |
| 30-60% | 轻微混乱，物品偶尔移动 |
| 60-80% | 中等混乱，记忆过期加快 |
| 80-100% | 高度混乱，频繁事件触发 |

### 2.3 混乱值来源

| 来源 | 增加量 |
|------|-------|
| 错误操作 | +5% |
| 重复搜索 | +3% |
| 记忆过期未更新 | +10% |
| 脚本事件触发 | +15% |

### 2.4 混乱值降低

| 方式 | 降低量 |
|------|-------|
| 成功放置物品 | -5% |
| 完成目标 | -20% |
| 正确回答Probe | -10% |

## 3. 交互系统

### 3.1 核心交互

| 操作 | 按键 | 描述 |
|------|------|------|
| 移动 | WASD/方向键 | 机器人移动 |
| 调整视角 | 鼠标拖动 | 旋转视角 |
| 缩放视野 | 鼠标滚轮 | 调整视野距离 |
| 切换视角 | V | 第一人称/俯视切换 |
| 拾取/放置 | F | 拾取物品或放入容器 |
| 打开/关闭 | F | 打开抽屉或柜门 |
| 保存记忆 | E | 保存附近物品记忆 |

### 3.2 碰撞检测

- 房间边界检测
- 家具碰撞检测
- 滑动碰撞（沿墙滑动避免卡住）

### 3.3 交互反馈

- **视觉反馈**：粒子效果、高亮、动画
- **音效反馈**：拾取、放置、打开、关闭音效
- **UI反馈**：Toast提示、得分变化、进度更新

## 5. 脚本事件系统

### 5.1 事件类型

| 类型 | 描述 | 触发条件 |
|------|------|---------|
| 钥匙猫 | 移动钥匙位置 | 步数触发 |
| 餐盘精 | 把脏盘子放回桌子 | 时间触发 |
| 袜子幽灵 | 交换篮子里的袜子 | 条件触发 |
| 电话响 | 提示玩家接电话 | 时间触发 |
| 时间循环 | 物品回到初始位置 | 终局条件 |

### 5.2 事件触发机制

1. 事件按步数或时间触发
2. 事件可以有前置条件（如特定物品位置）
3. 事件触发时增加混乱值
4. 事件可以标记记忆为过期

## 6. 目标系统

### 6.1 目标类型

| 类型 | 描述 |
|------|------|
| 里程碑 | 达成后保持完成状态 |
| 终局约束 | 结算时必须仍成立 |

### 6.2 目标判定

- 目标有依赖关系
- 依赖目标达成后才判定当前目标
- 目标通过谓词函数判定
- 达成目标时播放庆祝效果

## 7. 游戏流程

```
首页 → 任务选择 → 任务简报 → 游戏进行 → 记忆Probe → 结果分析 → 返回任务选择
```

### 7.1 游戏阶段

| 阶段 | 描述 |
|------|------|
| idle | 空闲状态 |
| briefing | 任务简报 |
| playing | 游戏进行中 |
| probing | 记忆测试 |
| analyzing | 分析中 |
| result | 结果展示 |
| aborted | 任务中止 |

### 7.2 胜利条件

- 完成所有目标
- 在时间限制内完成（如果有）

### 7.3 失败条件

| 条件 | 描述 |
|------|------|
| 混乱值满 | 系统完全失控 |
| 时间耗尽 | 未在时限内完成 |
| Probe失败 | 记忆测试正确率太低 |
