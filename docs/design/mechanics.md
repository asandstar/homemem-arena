# 记忆宅邸：失忆管家 - 核心机制

## 1. 记忆系统

### 1.1 记忆槽

玩家拥有 3 个记忆槽，每个记忆槽可以保存一件物品的位置信息。

```
记忆槽状态：
├── 空槽 → 可保存新记忆
├── 已保存 → 显示物品名称和位置
├── 已过期 → 显示红色警告，记忆可能不准确
└── 已锁定 → 记忆不会过期
```

### 1.2 记忆类型

| 类型 | 图标 | 描述 |
|------|------|------|
| 空间记忆 | 📍 | 记录物品所在房间和位置 |
| 物体记忆 | 📦 | 记录物品的属性和类别 |
| 时间记忆 | ⏱️ | 记录物品被移动的时间 |
| 程序记忆 | 🔄 | 记录操作序列和流程 |

### 1.3 记忆操作

| 操作 | 按键 | 描述 |
|------|------|------|
| 保存记忆 | E | 保存附近物品的记忆 |
| 更新记忆 | E | 更新已保存物品的记忆 |
| 锁定记忆 | 点击锁图标 | 锁定记忆防止过期 |
| 删除记忆 | 点击垃圾桶 | 删除记忆释放槽位 |

### 1.4 记忆过期机制

- 记忆有置信度值（0-100%）
- 置信度随时间下降
- 混乱值越高，置信度下降越快
- 置信度低于阈值时标记为过期
- 过期记忆可能包含错误信息

## 2. 混乱值系统

### 2.1 混乱值定义

混乱值（0-100%）代表系统的失控程度。

### 2.2 混乱值影响

| 混乱值范围 | 效果 |
|-----------|------|
| 0-30% | 正常状态，物品稳定 |
| 30-60% | 轻微混乱，物品偶尔移动 |
| 60-80% | 中等混乱，记忆过期加快 |
| 80-100% | 高度混乱，频繁事件触发 |

### 2.3 混乱值来源

| 来源 | 增加量 |
|------|-------|
| 错误操作 | +5% |
| 重复搜索 | +3% |
| 记忆过期未更新 | +10% |
| 脚本事件触发 | +15% |

### 2.4 混乱值降低

| 方式 | 降低量 |
|------|-------|
| 成功放置物品 | -5% |
| 完成目标 | -20% |
| 正确回答Probe | -10% |

## 3. 交互系统

### 3.1 核心交互

| 操作 | 按键 | 描述 |
|------|------|------|

| 缩放视野 | 鼠标滚轮 | 调整视野距离 |
| 切换视角 | V | 第一人称/俯视切换 |
| 拾取/放置 | F | 拾取物品或放入容器 |
| 打开/关闭 | F | 打开抽屉或柜门 |
| 保存记忆 | E | 保存附近物品记忆 |

### 3.2 碰撞检测

- 房间边界检测
- 家具碰撞检测
- 滑动碰撞（沿墙滑动避免卡住）

### 3.3 交互反馈

- **视觉反馈**：粒子效果、高亮、动画
- **音效反馈**：拾取、放置、打开、关闭音效
- **UI反馈**：Toast提示、得分变化、进度更新
- **对话反馈**：NPC对话、剧情推进

## 5. 脚本事件系统

### 5.1 事件类型

| 类型 | 描述 | 触发条件 |
|------|------|---------|
| 钥匙猫 | 移动钥匙位置 | 步数触发 |
| 餐盘精 | 把脏盘子放回桌子 | 时间触发 |
| 袜子幽灵 | 交换篮子里的袜子 | 条件触发 |
| 电话响 | 提示玩家接电话 | 时间触发 |
| 时间循环 | 物品回到初始位置 | 终局条件 |

### 5.2 事件触发机制

1. 事件按步数或时间触发
2. 事件可以有前置条件（如特定物品位置）
3. 事件触发时增加混乱值
4. 事件可以标记记忆为过期
5. 事件触发时播放对话

## 6. 目标系统

### 6.1 目标类型

| 类型 | 描述 |
|------|------|
| 里程碑 | 达成后保持完成状态 |
| 终局约束 | 结算时必须仍成立 |

### 6.2 目标判定

- 目标有依赖关系
- 依赖目标达成后才判定当前目标
- 目标通过谓词函数判定
- 达成目标时播放庆祝效果和对话

## 7. 对话系统

### 7.1 对话触发机制

对话通过以下方式触发：

| 触发类型 | 触发条件 | 示例 |
|---------|---------|------|
| `start` | 关卡开始 | `start_task-clean-table` |
| `roomEnter` | 进入房间 | `roomEnter_living` |
| `event` | 特定事件触发 | `memory_save`、`rank_s` |
| `goalComplete` | 目标达成 | `goalComplete_g-dirty-cup` |

### 7.2 对话节点结构

```typescript
{
  id: string,              // 节点唯一ID
  speaker: string,         // 说话者类型: 'system' | 'character' | 'narrator'
  speakerName: string,     // 说话者名称
  text: string,            // 对话内容
  choices?: DialogChoice[], // 可选分支
  autoContinue?: boolean,  // 是否自动继续
  autoContinueDelay?: number, // 自动继续延迟(毫秒)
}
```

### 7.3 对话选项效果

| 效果类型 | 描述 | 值 |
|---------|------|------|
| `score` | 增加分数 | 数值 |
| `hint` | 显示提示 | 提示文本 |

### 7.4 对话优先级

对话按优先级排序，高优先级对话优先显示：

| 优先级 | 类型 |
|--------|------|
| 15-20 | 关卡完成、评级对话 |
| 10 | 关卡开场、关键事件 |
| 8 | 目标完成反馈 |
| 6-7 | 记忆保存/过期 |
| 5 | 进入房间 |
| 4 | 放错位置提示 |

## 8. 角色设定

### 8.1 MEM-07（玩家）

- **身份**：记忆模块出了故障的家务机器人
- **性格**：冷静、数据化、略带幽默
- **记忆限制**：只能同时记住 3 件物品
- **职责**：完成宅邸中的各种家务任务
- **对话风格**：使用数据和技术术语，但也会给出鼓励

### 8.2 钥匙猫

- **身份**：宅邸的守护者，喜欢收集钥匙
- **性格**：调皮、爱恶作剧、好奇心强
- **能力**：可以移动钥匙位置
- **对话风格**：用"喵~"结尾，喜欢挑战玩家

### 8.3 餐盘精

- **身份**：负责餐桌事务的小精灵
- **性格**：得意、爱捣乱、不服输
- **能力**：可以把脏盘子放回桌上
- **对话风格**：笑声多，喜欢看玩家手忙脚乱

### 8.4 袜子幽灵

- **身份**：洗衣房的幽灵
- **性格**：害羞、爱藏东西、有点胆小
- **能力**：可以交换篮子位置、藏起袜子
- **对话风格**：说话断断续续，喜欢躲躲闪闪

### 8.5 早餐闹钟

- **身份**：厨房的闹钟，陷入了时间循环
- **性格**：强迫症、固执、机械
- **能力**：可以触发时间循环、移动物品
- **对话风格**："嘀嘀嘀！"开头，强调流程正确性

### 8.6 记忆宅邸（旁白）

- **身份**：宅邸本身的叙述者
- **作用**：营造氛围、描述场景变化
- **对话风格**：诗意、神秘、略带悬疑

## 9. 游戏流程

```
首页 → 任务选择 → 任务简报 → 游戏进行 → 记忆Probe → 结果分析 → 返回任务选择
```

### 9.1 游戏阶段

| 阶段 | 描述 |
|------|------|
| idle | 空闲状态 |
| briefing | 任务简报 |
| playing | 游戏进行中 |
| probing | 记忆测试 |
| analyzing | 分析中 |
| result | 结果展示 |
| aborted | 任务中止 |

### 9.2 胜利条件

- 完成所有目标
- 在时间限制内完成（如果有）

### 9.3 失败条件

| 条件 | 描述 |
|------|------|
| 混乱值满 | 系统完全失控 |
| 时间耗尽 | 未在时限内完成 |
| Probe失败 | 记忆测试正确率太低 |

## 10. 存档系统

### 10.1 存档内容

| 内容 | 描述 |
|------|------|
| 关卡进度 | 已解锁、已完成的关卡 |
| 星级评价 | 各关卡的评级 |
| 最佳分数 | 各关卡的最高分 |
| 游戏设置 | 音效开关、控制偏好 |

### 10.2 自动保存

- 关卡完成后自动保存
- 每 30 秒自动保存一次
- 切换关卡时自动保存

### 10.3 存档格式

使用 localStorage 存储 JSON 格式数据。
