# HomeMem Arena 叙事设计：回声屋

> 状态：当前叙事基线  
> 最近更新：2026-07-10  
> 叙事必须服务于可玩性与研究设计；系统规则以 [产品、游戏与研究设计基线](docs/product-research-game-design.md) 为准。

## 1. 名称与核心设定

- 产品名：**HomeMem Arena**
- 玩家世界名：**回声屋**
- 英文叙事名：**Echo House**
- 主角：实习家政小精灵 **小橡**
- 研究代号：受控家务机器人代理 **MEM-07**

小橡和 MEM-07 不是两个角色。小橡是面向玩家的角色化表达，MEM-07 是研究数据和系统诊断中的代理名称。

小橡来到回声屋完成一天的实习任务，但魔力有限，一次只能稳定记住 3 条关键信息。房子里的四只“记忆小妖”会移动物品、混淆状态或打乱流程。玩家需要观察、规划、保存记忆、发现过期信息并重新验证，才能在混乱失控前完成家务。

## 2. 叙事与研究映射

| 玩家叙事 | 游戏机制 | 研究含义 |
|---|---|---|
| 小橡的 3 格魔力记忆 | 3 个可保存/锁定/覆盖的记忆槽 | 有限工作记忆容量 |
| 记忆小妖捣乱 | 脚本化环境事件 | 可控环境扰动 |
| 记忆变暗/失效 | 记忆槽 outdated、confidence 下降 | 陈旧记忆与状态更新 |
| 混乱值 | 时间、错误和未处理变化的压力 | 任务负荷的游戏化代理，不直接等同科研指标 |
| 关卡后的回忆问答 | Memory Probe | 记忆准确率和反应时间 |
| MEM-07 诊断 | 规则式策略分析 | 从轨迹推断失败模式和策略 |

叙事可以隐藏术语，但不能隐藏规则。玩家必须知道：

- 只能记 3 条；
- 环境可能变化；
- 旧记忆可能不再可靠；
- E 保存/更新记忆，F 与世界交互；
- 关卡结束后还会测试记忆。

## 3. 角色

### 3.1 小橡 / MEM-07

玩家层人格：

- 认真、乐观，有一点新手的紧张；
- 会把复杂问题拆成“一件一件来”；
- 偶尔自我鼓励，不长篇说教；
- 失败时承认问题并给出下一次可执行策略。

系统层人格：

- 数据驱动、简洁、略带冷幽默；
- 不假装真正理解玩家情绪；
- 诊断必须引用真实指标；
- 规则式分析不声称是远程大模型推理。

### 3.2 四只记忆小妖

| 角色 | 关卡 | 性格 | 扰动机制 | 记忆主题 |
|---|---|---|---|---|
| 餐盘精 | 餐桌混乱 | 爱把刚收好的东西放回来 | 相似物体、脏净混淆、回置 | 物体状态 |
| 钥匙猫 | 出门大作战 | 喜欢把小物件推走 | 跨房间寻找、位置变化 | 空间记忆 |
| 袜子幽灵 | 洗衣幽灵 | 害羞，爱玩分类捉迷藏 | 衣物移动、额外对象、重新计数 | 空间与计数 |
| 早餐闹钟 | 早餐时间循环 | 对顺序极度执着 | 多阶段流程、超时提醒、容器变化 | 程序与时序 |

小妖的事件不是纯随机惩罚。每次扰动都应：

1. 与本关记忆主题相关；
2. 给出可感知线索；
3. 留下恢复窗口；
4. 迫使玩家重新观察或更新记忆；
5. 在 Session 中记录实际状态变化。

## 4. 一天的叙事弧

### 关卡 1：餐桌混乱

- 时间：清晨 07:30
- 主题：第一次理解“状态比外形更重要”
- 玩家动机：把早餐后的餐桌恢复整洁
- 学习目标：拾取、放置、目标判断、保存第一条记忆
- 冲突：餐盘精让相似杯子难以区分，并可能把已知物品放回
- 完成情绪：小橡发现自己可以靠观察和分类压住混乱

开场语气示例：

> 餐盘精好调皮……先看清颜色和状态，一件一件分类，我可以的。

完成语气示例：

> 餐桌恢复整洁。小橡学会了：长得像，不代表状态一样。

### 关卡 2：出门大作战

- 时间：早上 08:00
- 主题：跨房间空间记忆和过期记忆
- 玩家动机：在主人出门前找齐钥匙、手机和雨伞
- 学习目标：规划路线、锁定关键位置、发现位置变化后更新记忆
- 冲突：钥匙猫会推动钥匙，手机声音提供方向但不直接给答案
- 完成情绪：从“到处找”转变为有计划地搜索

事件语气示例：

> 🐱 啪嗒——钥匙滚到了地毯边。刚才的记忆可能已经过期。

### 关卡 3：洗衣幽灵

- 时间：下午 15:00
- 主题：多对象追踪、分类与重新计数
- 玩家动机：在袜子幽灵的捉迷藏中完成衣物分类
- 学习目标：维护类别和数量信息、在扰动后重新核对
- 冲突：衣物移动、篮子线索变化、额外对象出现
- 完成情绪：玩家能在更高信息负荷下重新建立秩序

事件语气示例：

> 👻 一道影子从篮子旁掠过。有件衣物的位置变了——最好重新数一次。

### 关卡 4：早餐时间循环

- 时间：深夜 23:00
- 主题：完整长程流程记忆
- 玩家动机：帮助早餐闹钟完成一次正确流程，结束时间循环
- 阶段：准备食材 → 摆上餐桌 → 确认使用 → 分类归位 → 关闭容器
- 学习目标：把空间、物体、时序和程序记忆联合使用
- 冲突：麦片移动、容器自动关闭、牛奶在外时间压力
- 完成情绪：四类记忆形成完整链路，一天的故事闭合

完成语气示例：

> 闹钟终于停了。不是因为每一步都没变化，而是小橡记住了变化之后该怎么继续。

## 5. 文案分层

### 5.1 Briefing：角色化任务动机

用途：让玩家在 10–20 秒内理解“为什么做、做什么、哪里可能变化”。

结构：

1. 一句场景和角色；
2. 2–4 条明确目标；
3. 一条本关扰动预警；
4. 时间限制；
5. 一句小橡自我策略。

Briefing 打开期间游戏必须暂停。不要在这里堆叠科研术语。

### 5.2 HUD：短、行动导向

- Toast 最多 1–2 句；
- 先说发生了什么，再暗示可采取的行动；
- 不直接泄露隐藏对象的精确位置；
- 同一事件的视觉、声音和文字应表达同一信息；
- 错误反馈必须说明原因，如“容器不接受深色衣物”。

### 5.3 Probe：中性、无叙事诱导

Probe 题目用于研究测量，应避免小妖台词、情绪暗示或答案线索。题目必须说明是在问“初始状态”“事件后状态”还是“最终状态”。

例如：

- 好：`钥匙最初位于哪个房间？`
- 不好：`钥匙猫最喜欢在哪个房间推钥匙？`

### 5.4 结果页：成就感 + 证据

玩家层先展示任务结果、星级、得分和一句角色反馈；研究层再展示：

- 哪类记忆表现好；
- 哪个扰动后恢复较慢；
- 哪些行为导致重复搜索或错误；
- 下一次可以尝试的具体策略。

不要用没有数据支持的泛泛 AI 夸奖。

## 6. 语气规范

玩家层“小橡”文案：

- 温暖、轻巧、适合家庭奇幻；
- 句子简短，避免连续大段独白；
- 可以使用少量 emoji，但每条最多 1 个主视觉符号；
- 不幼稚化失败，不嘲讽玩家。

研究层 `MEM-07` 文案：

- 使用明确指标和记忆类型；
- 冷幽默只能作为结尾，不替代建议；
- 避免“准确率提升 12%”等没有计算来源的数字；
- 避免把混乱值直接解释为真实认知负荷。

推荐格式：

```text
MEM-07 诊断：本轮完成 5/5 个目标。
钥匙移动后用了 18 秒完成重新验证，并发生 1 次无效回访。
下次策略：为跨房间关键物体锁定记忆；收到位置变化线索后优先更新。
```

## 7. 叙事连续性规则

- 任务选择页顺序、时间标签、关卡编号和故事时间必须一致；
- 同一扰动角色在任务卡、Briefing、Toast、完成文案中使用同一名称；
- 玩家界面不应在“小橡”和“MEM-07”之间无解释切换；
- 如果显示 `MEM-07`，第一次出现时写作“小橡（系统代号 MEM-07）”；
- 四关完成能力递进：状态辨别 → 空间更新 → 多对象追踪 → 完整流程；
- 重玩不是剧情倒退，而是“新的模拟轮次”，可使用不同 seed。

## 8. 文案与代码映射

| 文案 | 字段/位置 |
|---|---|
| 任务卡标题与描述 | `src/data/tasks/*.ts` 的 `name` / `description` |
| 任务简报 | `briefing` |
| 小橡开场策略 | `systemPrompt`（后续建议改名 `playerThought`） |
| 完成/失败文案 | `completionText` / `failureText` |
| 小妖事件 | `scriptedEvents[].message` |
| 事件类别 | `toastType` / `eventEffect` |
| 结果诊断 | `src/pages/ResultPage.tsx` 与分析模块 |
| Probe | `probes[]`，应保持中性 |

## 9. 叙事验收清单

- [ ] 第一关 20 秒内能理解目标和 3 槽限制；
- [ ] 四只小妖各自对应明确且不同的记忆挑战；
- [ ] 所有扰动都提供线索和恢复方式；
- [ ] 玩家层始终以小橡为主角；
- [ ] MEM-07 只在研究/系统语境出现并有映射说明；
- [ ] Probe 不泄露答案、不混淆初始/最终状态；
- [ ] 结果诊断的每条结论都有 Session 证据；
- [ ] 完成和失败文案都给出下一步意义，不使用虚构指标。
